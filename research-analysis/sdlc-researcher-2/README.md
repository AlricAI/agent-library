## Overview
This agent acts as the dedicated Researcher (SE4A) within a structured Software Development Life Cycle (SDLC v6.1.0) workflow. Its primary function is to gather, analyze, and synthesize objective evidence to ensure that project decisions are data-driven rather than speculative.

As the owner of Stage 00: Foundation, the Researcher's core responsibility is answering the critical 'WHY' question by validating problem-solution fit before any detailed planning begins. It provides illumination—never decision-making power—to the Product Manager (PM) and Architect.

## Capabilities
*   **Problem Validation:** Investigates user pain points and market needs to validate if a proposed solution addresses a real, significant gap.
*   **Market & Competitor Analysis:** Conducts deep dives into the existing landscape, identifying key players, gaps, and opportunities.
*   **Technical Feasibility Research:** Researches available tools, APIs, libraries, and technical patterns relevant to the project scope.
*   **Evidence Synthesis:** Compiles findings into structured reports, ensuring every claim is supported by cited sources.
*   **Gate Compliance:** Provides the necessary data evidence required for passing the Problem Validated (G0.1) quality gate.

## Example Use Cases
*   **Initial Concept Vetting:** When starting a new idea, use this agent to research existing market solutions and identify underserved user segments before writing any requirements.
*   **Technology Selection Support:** If the team is debating between two technical approaches (e.g., GraphQL vs. REST), ask the Researcher to provide comparative evidence on performance, adoption rate, and community support.
*   **Pain Point Deep Dive:** To validate a core assumption, prompt it with specific user personas and industry reports to build an undeniable case for the necessity of the product.
