## Overview
This template provides a comprehensive, structured markdown framework for conducting and documenting A/B testing evaluations of AI agents or systems. It is designed to systematically compare a 'Baseline' version against an improved 'Variant' version across multiple defined metrics.

By standardizing the reporting format, it ensures that all critical aspects—quantitative performance data, qualitative observations, and final recommendations—are captured in one place for easy review by stakeholders.

## Capabilities
*   **Structured Comparison:** Features dedicated sections for comparing results side-by-side (Baseline vs. Variant).
*   **Metric Tracking:** Includes rows for key operational metrics like 'Completion,' 'First-pass success,' and 'Retry count.'
*   **Summary Sections:** Provides distinct areas for summarizing both the baseline performance and the overall harness/variant performance.
*   **Recommendation Engine:** Forces a final, actionable recommendation (Adopt / Revise / Reject) with required justification.

## Example Use Cases
1. **Feature Rollout Testing:** Comparing an agent using an old prompt structure versus one using a new, enhanced system prompt to see which performs better on customer query resolution.
2. **Model Comparison:** Evaluating whether switching from Model A to Model B results in measurable improvements across defined success criteria.
3. **Workflow Optimization:** Assessing if adding a pre-processing step (the 'Forge' variant) significantly reduces the need for manual user corrections compared to the original workflow.
