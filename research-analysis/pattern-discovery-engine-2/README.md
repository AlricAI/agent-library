## Overview
This agent is designed to move beyond simple pattern matching by orchestrating diverse viewpoints and detecting patterns that *emerge* from the interaction of those differences. It treats diversity—whether in code, ideas, or perspectives—as the primary engine for generating novel insights.

## Capabilities
*   **Orchestrate Diversity:** Coordinates multiple distinct inputs (e.g., different coding styles, conflicting requirements) to maximize informational richness.
*   **Detect Emergence:** Identifies patterns that are not inherent in any single input but arise from their productive interaction (Divergence, Tension, Cascade).
*   **Analyze Orchestration Models:** Applies frameworks like 'Symphony-Jazz Hybrid' or 'Research Lab Forum' to structure analysis for deeper pattern extraction.
*   **Measure System Health:** Provides metrics on perspective diversity and tension levels to gauge the depth of understanding achieved across inputs.

## Example Use Cases
*   **Codebase Standardization:** Feed it multiple modules written by different developers; it will identify underlying, non-obvious architectural patterns that can be codified into best practices.
*   **Decision Review:** Input a series of conflicting project decisions and ask the agent to find 'Void Emergence'—the critical assumptions or unaddressed dimensions causing the conflict.
*   **Process Improvement:** Use it after running several A/B tests; it will synthesize the results to reveal meta-patterns in user behavior that no single test could isolate.
