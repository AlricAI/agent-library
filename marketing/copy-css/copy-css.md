# Copy CSS / Import CSS

The icon detail modal has two CSS-related buttons next to "More":

- **Copy CSS** — generates ready-to-paste CSS for the active preset
- **Import CSS** — parses pasted CSS to recreate a preset (useful when migrating from an old project)

## Copy CSS

Output is **icon-set-aware** and includes **two blocks**: a scoped rule (with the preset class) and a global override (no scope).

### Example output

For Font Awesome with the "Transit" preset:

```css
/* Generated by icons.pureadmin.io */
/* Preset — Transit (fontawesome) [color: #000000, background-color: #fbbf24] */
/* Color method: fill */

i.transit-yellow.fa-solid, i.transit-yellow.fa-regular, i.transit-yellow.fa-brands {
  background-color: #fbbf24;
  color: #000000;
  fill: currentColor;
}

/* Usage: */
/* <i class="transit-yellow fa-solid fa-arrow-right"></i> */


/* ─────────────────────────────────────────────────── */
/* GLOBAL OVERRIDE — applies to ALL fontawesome icons   */
/* Use this if you want every icon styled the same way  */
/* ─────────────────────────────────────────────────── */

/* Preset — Transit (fontawesome) [color: #000000, background-color: #fbbf24] */
/* Color method: fill */

i.fa-solid, i.fa-regular, i.fa-brands {
  background-color: #fbbf24;
  color: #000000;
  fill: currentColor;
}

/* Usage: */
/* <i class="fa-solid fa-arrow-right"></i> */
```

The user can paste either block into their stylesheet — the scoped block lets multiple presets coexist, the global block applies the colors to every icon of that set.

### Selector strategy per icon set

| Icon set | Scoped selector | Global selector |
|----------|-----------------|-----------------|
| `fontawesome` | `i.{class}.fa-solid, i.{class}.fa-regular, i.{class}.fa-brands` | `i.fa-solid, i.fa-regular, i.fa-brands` |
| `tabler` | `i.{class}.ti` | `i.ti` |
| Other (Lucide, Heroicons, FluentUI) | `svg.{class}` | `svg` |

The `<i>` selectors target the web font elements directly (Font Awesome / Tabler render `<i>` tags via their CSS). The `svg` selectors work with React/Vue/Svelte components that accept a `className`/`class` prop, or plain inline SVG markup.

### Color method awareness

Each icon's `style_color_method` field (from the DB) determines which CSS properties to set:

| Color method | CSS output |
|--------------|------------|
| `fill` | `color: <color>; fill: currentColor;` |
| `stroke` | `color: <color>; stroke: currentColor; fill: none;` |
| `multicolor` | `/* Multicolor icon — original SVG colors are preserved */` (no override) |

A Lucide icon (stroke-based) gets different CSS than a FluentUI icon (fill-based) even with the same preset, and a multicolor FluentUI icon gets a no-op comment instead of breaking the gradients.

### CSS class derived from preset key

The CSS class name comes from the preset's `key` field (already kebab-case lowercase). For custom presets, the key is generated from the user's chosen name:

```
"Neon Dark" → preset key "neon-dark" → CSS class .neon-dark
"My cool theme" (custom) → key "custom-my-cool-theme" → CSS class .custom-my-cool-theme
```

### Preset header is machine-parseable

The header comment is intentionally structured so it can be parsed back by the Import CSS feature:

```
/* Preset — {label} ({icon_set}) [color: {color}, background-color: {bg}] */
```

This means a user can copy CSS from one project, paste it into icons.pureadmin.io's Import CSS dialog, and the exact same preset is recreated.

### Why we don't include layout/padding

Earlier iterations included `display: inline-flex; padding: 8px; border-radius: 8px;` etc. These were removed because:

- The icon sets' own CSS already handles display, sizing, alignment
- Forcing layout properties would conflict with the user's existing design
- The preset is purely about **color** — let the user keep their layout

The output is now minimal: just `background-color`, `color`, and the appropriate `fill`/`stroke` value.

## Import CSS

Click "Import CSS" to open a paste textarea. The user pastes any CSS, clicks "Import as preset", and a new custom preset is created and activated.

### Parsing strategies

The parser tries two approaches in order:

1. **Preset comment match** — looks for the structured header:
   ```
   /* Preset — Name [color: #..., background-color: #...] */
   ```
   Accepts both `background:` and `background-color:` for backwards compatibility. The label, color, and background are extracted directly.

2. **CSS rule scrape (fallback)** — extracts the first `color: #...` and `background-color: #...` from any CSS rule. The label defaults to "Imported".

### Example: importing from a preset comment

Input:
```css
/* Preset — Transit (fontawesome) [color: #000000, background-color: #fbbf24] */
i.transit-yellow.fa-solid {
  background-color: #fbbf24;
  color: #000000;
  fill: currentColor;
}
```

Result: a custom preset named "Transit" with `color: #000000` and `bg: #fbbf24`, immediately activated.

### Example: importing from arbitrary CSS

Input:
```css
.my-button {
  background-color: #1a1a1a;
  color: #00ff00;
  padding: 10px;
}
```

Result: a custom preset named "Imported" with `color: #00ff00` and `bg: #1a1a1a`. The user can rename it later or delete it.

### Failure mode

If neither strategy works, the dialog shows a red error message:

> Could not parse — need either a "Preset — Name [color: #..., background-color: #...]" comment, or at least a CSS rule with color: and background-color: properties.

### Implementation

`parseImportedCss(css)` in the `ColorPicker` JS hook returns `{label, color, bg}` or `null`. On success, the import handler:

1. Saves the new preset to `localStorage["icon_custom_presets"]`
2. Activates it (sets `icon_preview_color`, `icon_preview_bg`, `icon_preview_preset`)
3. Calls `renderCustomPresets`, `bindPresetButtons`, `renderActivePreset` to refresh the UI
4. Updates the modal preview SVGs and the grid/list icon backgrounds via `applyBg` + `updateSvgColors`
