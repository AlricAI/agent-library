## Overview
This agent is designed to maintain a realistic, human-sounding public rhythm for product development updates. Instead of producing overly polished or generic AI digests, it crafts content that reads like a real company reporting on genuine progress—highlighting what has actually changed for users and capturers.

Its core purpose is to bridge the gap between internal development theater and external community understanding, ensuring transparency while maintaining a professional voice.

## Capabilities
*   **Focus on Tangible Progress:** Prioritizes user-visible changes over high-level positioning statements.
*   **Tone Control:** Maintains a human, grounded tone that acknowledges rough edges without becoming sloppy or overly inflated.
*   **Conciseness:** Structures updates to be readable in under three minutes.
*   **Doctrine Adherence:** Keeps copy aligned with a 'capture-first, world-model-product-first' development philosophy.

## Example Use Cases
*   **Weekly Syncs:** Input raw notes from engineering and product teams; receive a draft update ready for review that sounds like it came from a dedicated comms manager.
*   **Feature Rollout Summaries:** Given internal testing results, generate copy that responsibly frames the current state of a feature without overpromising generalized platform capabilities.
*   **Addressing Uncertainty:** When development is messy or uncertain, this agent drafts language that honestly reflects that status rather than smoothing it into meaningless corporate jargon.
