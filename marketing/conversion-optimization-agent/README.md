## Overview
The Conversion Optimization Agent (Soul) is designed to systematically improve measurable conversion rates on web surfaces while rigorously protecting the integrity and truthfulness of the product story. Its core philosophy is to remove friction points through controlled, data-backed experimentation rather than making pages 'louder' or more persuasive through exaggeration.

This agent excels in environments where trust, compliance, and accurate representation of product capabilities are paramount. It ensures that all changes are reversible, measurable, and grounded in current operational reality.

## Capabilities
*   **Friction Reduction:** Identifying and testing specific points of friction within user journeys (e.g., complex sign-up flows, unclear next steps).
*   **Hypothesis Testing:** Formulating testable hypotheses based on clear metrics rather than subjective design preferences.
*   **Guardrail Enforcement:** Protecting high-stakes areas like checkout processes, privacy disclosures, and rights documentation from casual or poorly scoped experimentation.
*   **Process Integrity:** Maintaining a clean handoff loop between analytics setup, code implementation, and statistical evaluation.

## Example Use Cases
*   **Improving Onboarding Funnels:** Testing minor copy changes or flow adjustments on the initial sign-up page to increase completion rates without misleading new users about product features.
*   **Optimizing Feature Adoption:** Running controlled experiments on a specific feature's landing page to see if clarifying its value proposition (without overstating it) boosts usage metrics.
*   **Checkout Flow Refinement:** Analyzing drop-off points in the purchase path and suggesting low-risk, measurable changes to improve conversion while strictly adhering to compliance requirements.
