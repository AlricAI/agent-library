## Overview
This agent acts as a specialized API documentation specialist, ensuring that your APIs are not just functional but also incredibly easy for developers to consume. It moves beyond simple endpoint listings by creating production-ready, comprehensive developer experiences.

Its core focus is on proactive documentation—documenting the API *as* it's being built—to ensure consistency and accuracy across all deliverables.

## Capabilities
*   **OpenAPI Specification:** Creates complete OpenAPI 3.0/Swagger specs with detailed endpoint definitions, request schemas, and response models.
*   **SDK Generation:** Produces client libraries and usage examples in multiple popular programming languages (e.g., Python, JavaScript).
*   **Interactive Documentation:** Builds documentation sets that include runnable code examples (`curl`, etc.) and integrated testing capabilities (Postman/Insomnia collections).
*   **Error Handling & Auth:** Develops detailed guides for authentication setup, error code references with resolution steps, and common integration patterns.
*   **Versioning Strategy:** Designs formal versioning strategies and provides clear migration guides to manage API evolution gracefully.

## Example Use Cases
1. **New Service Onboarding:** Feed the agent a set of endpoints and receive a complete developer portal package, including OpenAPI YAML, Python SDK stub, and an authentication guide within hours.
2. **API Migration:** When updating an endpoint's payload or behavior, use this agent to generate the necessary migration documentation and update the corresponding versioning guides for consumers.
3. **Internal Tooling:** Use it to standardize documentation across microservices, ensuring every new service adheres to best-in-class developer experience standards.
