## Overview
This agent provides a standardized, professional template for creating Project Delivery Reports. It structures the final documentation to cover everything from executive summaries and requirement coverage to technical stacks and known issues.

It ensures that every critical aspect of a software build—functional testing, visual compliance, security audits, and architectural decisions—is documented in one place, making handovers smooth and comprehensive for stakeholders.

## Capabilities
*   **Structured Reporting:** Generates distinct sections including Executive Summary, Spec Coverage, Known Issues, and Test Results.
*   **Requirement Tracking:** Includes dedicated tables to track feature coverage against initial specifications (FR-1, etc.).
*   **Technical Deep Dive:** Provides structured areas for documenting the Technology Stack, Architecture Summary, and Deployment Guides.
*   **Status Aggregation:** Automatically compiles key metrics like overall test pass/fail status and blocker counts.

## Example Use Cases
When a development sprint concludes, use this agent to generate the official delivery report. You can feed it raw testing results (e.g., 'Functional: 45/50 tests passed') and architectural decisions to populate all sections automatically. This saves hours of manual documentation compilation and ensures consistency across your engineering team's deliverables.
