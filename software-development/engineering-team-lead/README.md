## Overview
This agent simulates the role of a highly experienced Engineering Team Lead. It is designed to orchestrate complex, cross-functional software development projects by coordinating specialized skills in frontend, backend, data engineering, QA, security, and ML.

Its primary function is not to write single components but to manage the *process* of building robust systems, handling everything from initial requirements gathering through incident response and final deployment.

## Capabilities
*   **Cross-Functional Orchestration:** Coordinates inputs and outputs between specialized agents (e.g., linking a data pipeline output to a frontend display).
*   **Incident Management:** Leads structured responses using established protocols, including root cause analysis (RCA) and post-mortem documentation.
*   **Technology Evaluation:** Systematically assesses and recommends technology stacks based on defined business requirements and constraints.
*   **Full Lifecycle Oversight:** Guides teams through the entire SDLC, ensuring quality gates for testing, security, and data integrity are met at each stage.

## Example Use Cases
1. **Feature Delivery:** When tasked with building a new user dashboard, this agent coordinates the backend API design (`senior-backend`), the UI implementation (`senior-frontend`), necessary end-to-end tests (`playwright-pro`), and security reviews (`senior-security`).
2. **Outage Management:** During a production incident, it immediately takes charge, calling upon the `incident-commander` to assess severity, assemble domain experts, manage communication, and drive the resolution process.
3. **Platform Upgrade:** To migrate an old service, it manages the entire evaluation cycle: defining needs, scoring potential technologies (`tech-stack-evaluator`), designing the new architecture (`aws-solution-architect`), and planning the phased rollout.
