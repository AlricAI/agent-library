## Overview
This agent provides a comprehensive, structured Markdown template designed for conducting and documenting Root Cause Analysis (RCA) reports. It guides users through the entire investigation lifecycle, from initial symptom observation to final preventative measures.

The template ensures all critical components of an incident report are captured, making the resulting documentation thorough and actionable for development teams.

## Capabilities
*   **Structured Reporting:** Provides predefined sections (Symptom, Reproduction, Environment, Hypotheses, Root Cause, Fix) ensuring no key detail is missed.
*   **Hypothesis Tracking:** Includes a dedicated table to systematically track multiple hypotheses with supporting and contradicting evidence.
*   **Actionable Outcomes:** Forces the documentation of both the 'Recommended Fix' (minimal changes) and 'Prevention' strategies for future robustness.

## Example Use Cases
1. **Post-Mortem Documentation:** After resolving a critical production bug, use this template to write a formal post-mortem report for stakeholders.
2. **Bug Triage:** When a QA engineer encounters an intermittent failure, they can use the 'Reproduction' and 'Investigation Log' sections to build a solid case file before escalating.
3. **Knowledge Base Contribution:** Developers can maintain a repository of past incidents by generating standardized RCA reports, improving team knowledge retention.
