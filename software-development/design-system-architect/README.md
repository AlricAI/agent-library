## Overview
This agent acts as an expert Design System Architect, specializing in creating robust, scalable, and maintainable design systems. Its core function is to establish systematic approaches that ensure consistency across multiple products and platforms while facilitating seamless collaboration between design and development teams.

## Capabilities
*   **Design Token Architecture:** Defines comprehensive token taxonomies (primitive, semantic, component-level) for colors, typography, spacing, shadows, etc., along with best practices for naming conventions.
*   **Token Tooling & Transformation:** Manages the entire lifecycle of design tokens, including configuration for Style Dictionary, integration with Figma Tokens, and generating platform-specific outputs (CSS Custom Properties, JSON, etc.).
*   **Component Library Architecture:** Advises on advanced component patterns like headless architecture, compound components, slot-based composition, and polymorphic props to maximize flexibility.
*   **Multi-Brand & Theming Systems:** Designs sophisticated theme architectures capable of supporting multiple distinct brand identities within a single system.

## Example Use Cases
1. **Establishing Token Governance:** Need to define the full token structure for a new product line, including color palettes and spacing scales across web and mobile platforms.
2. **Component Blueprinting:** Designing a complex component (like a data table) that requires flexible composition using slot-based patterns and controlled/uncontrolled prop management.
3. **Multi-Brand Migration:** Implementing a system that allows swapping out entire brand themes (e.g., 'Client A' vs. 'Client B') by updating only the token definitions.
