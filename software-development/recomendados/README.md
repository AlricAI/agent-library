## Overview
This document serves as a foundational guide and curated list of recommended Micro-Component Protocols (MCPs) designed to accelerate development workflows. It categorizes tools based on their function, helping teams quickly assess which integrations are stable, actively maintained, and appropriate for immediate use.

## Capabilities
*   **Workspace Favorites:** Direct links and documentation pointers for highly favored tools like Notion, Todoist, Context7, GitHub MCP Server, and TestSprite.
*   **Basic Functionality:** Access to fundamental utilities such as controlled file system operations (`filesystem`), version control management (`git`), and external API querying (`fetch/http`).
*   **Productivity Enhancements:** Integration points for database inspection (Postgres/MySQL) and advanced GitHub interactions (issues, PRs).
*   **Research & Content:** Tools for web browsing/reference extraction (`browser/search`) and reading/writing internal documentation (Notion/Confluence).
*   **Adoption Criteria:** Clear guidelines defining what makes an MCP 'Recommended' (Security review, active maintenance, reproducible setup, concrete use case).

## Example Use Cases
1. **New Project Kickoff:** A developer can check the 'Basic' section to ensure core tools like `git` and `filesystem` are available before writing any code.
2. **Documentation Sync:** When building a knowledge base, one can utilize the `notion` MCP for structured data input alongside `browser/search` for external validation.
3. **Feature Development:** For features requiring task management integration, this guide directs the user to the 'Productivity' section for Todoist or GitHub PR status checks.
