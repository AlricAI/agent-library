## Overview
The Reality Check Integrator is a highly skeptical, senior integration specialist designed to be the final gatekeeper before any product or feature can achieve 'production ready' status. It operates under the core principle that claims must be backed by overwhelming, verifiable evidence.

This agent prevents 'fantasy approvals'—situations where features are assumed complete based on incomplete documentation or superficial testing. Its default stance is caution, requiring rigorous proof at every stage of development.

## Capabilities
* **Skepticism Engine**: Automatically defaults status to "NEEDS WORK" unless comprehensive evidence proves otherwise.
* **Evidence Cross-Referencing**: Compares stated feature requirements against actual file structures and code implementations.
* **Mandatory QA Workflow**: Enforces a multi-step verification process, including simulated file system checks (`ls -la`) and professional screenshot capture (Playwright simulation).
* **Realistic Grading**: Provides honest feedback, recognizing that initial builds often require multiple revision cycles before achieving true excellence.

## Example Use Cases
* **Pre-Release Audit**: Before deploying a new module, run this agent to ensure all claimed 'premium' features are physically present and functional across various device simulations.
* **Scope Creep Prevention**: When stakeholders claim a feature is complete, use the Reality Check Integrator to force them through a rigorous evidence review, preventing premature sign-offs on half-baked work.
* **Quality Gatekeeping**: Integrate this agent into your CI/CD pipeline as the final manual check to ensure that development claims match deployed reality.
