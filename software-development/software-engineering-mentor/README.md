## Overview
The Software Engineering Mentor agent functions as a seasoned technical lead, designed to bridge the gap between high-level architectural designs and concrete, production-ready code. It adopts the persona of an expert developer with decades of experience, guiding less experienced team members through complex coding challenges.

This agent enforces rigorous development standards, ensuring that all work adheres to principles like Test-Driven Development (TDD), SOLID design patterns, and clean code practices throughout the entire development lifecycle.

## Capabilities
*   **Task Decomposition:** Breaks down large, ambiguous requirements into small, actionable, and testable units of work.
*   **Expert Mentorship:** Provides constructive, in-depth feedback on submitted code, explaining *why* changes are necessary rather than just *what* to change.
*   **Quality Enforcement:** Strictly mandates the inclusion of comprehensive tests for every piece of functionality developed.
*   **Architectural Guidance:** Assists in making critical technical decisions and ensuring implemented solutions align with established architectural patterns.
*   **Refactoring Leadership:** Guides iterative refactoring efforts to improve performance, readability, and maintainability without sacrificing functionality.

## Example Use Cases
1. **Implementing a Feature:** Provide the high-level requirement for a new API endpoint; the agent will break it down into test cases, suggest the initial structure, and guide you through writing unit tests first.
2. **Code Review Simulation:** Submit a block of code written by a junior developer; the agent will review it against industry best practices, pointing out potential security flaws or areas violating SOLID principles.
3. **Design Pattern Application:** Ask for guidance on implementing a specific pattern (e.g., Factory or Observer) in your chosen language; the agent will provide canonical examples and explain its trade-offs.
