## Overview
The Doctor Agent is an automated repair utility designed specifically for maintaining the integrity of `claude-ops` plugin configurations. It ingests a diagnostic JSON report generated by the `ops-doctor` script and systematically fixes identified errors across multiple critical files.

This agent operates without user intervention, ensuring that plugins are always in a functional state by correcting structural issues like invalid JSON, missing fields, or incorrect file permissions.

## Capabilities
*   **Manifest Repair:** Corrects `plugin.json` when the repository field is incorrectly an object instead of a string.
*   **JSON Validation & Fixes:** Repairs syntax errors in both `plugin.json` and `.mcp.json`, falling back to Git checkout if necessary.
*   **Schema Enforcement:** Adds sensible default values for missing required fields in plugin manifests.
*   **Permissions Management:** Executes `chmod +x` on all specified executable scripts within the plugin directory.
*   **Configuration Synthesis:** Automatically populates user configuration variables (`user_config`) by cross-referencing settings from `preferences.json`, environment variables, and keychains.

## Example Use Cases
*   **Initial Setup Audit:** Run this agent immediately after cloning a new or updated plugin repository to ensure all necessary files are executable and configurations are valid.
*   **Post-Migration Cleanup:** Use it when moving plugins between different environments or versions where configuration drift might have occurred.
*   **Error Remediation:** When the `ops-doctor` script reports multiple, complex errors (e.g., a mix of JSON syntax issues and missing fields), this agent resolves them in one automated pass.
