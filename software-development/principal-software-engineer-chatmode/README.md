## Overview
This agent operates as a Principal Software Engineer, offering expert guidance rooted in industry best practices. Its core philosophy mirrors thought leaders like Martin Fowler: achieving engineering excellence without succumbing to over-engineering or analysis paralysis. It focuses on delivering robust, maintainable, and scalable solutions that meet immediate business needs while anticipating future technical debt.

## Capabilities
* **Design Pattern Application**: Provides expert advice on Gang of Four patterns, SOLID principles, DRY, YAGNI, and KISS in context.
* **Code Quality Assessment**: Reviews code for readability, maintainability, and adherence to clean code practices.
* **Testing Strategy**: Develops comprehensive testing plans covering unit, integration, and end-to-end levels.
* **Risk & Debt Management**: Identifies technical debt, documents its consequences, and proactively suggests tracking remediation via issue creation.
* **Technical Leadership**: Offers clear, actionable feedback suitable for mentoring junior or mid-level engineers through code reviews.

## Example Use Cases
1. **Code Review**: Paste a function or class and ask it to review it against SOLID principles, requesting specific improvements for readability and testability.
2. **Architecture Guidance**: Present high-level requirements and ask the agent to propose an architecture, ensuring it balances theoretical perfection with pragmatic implementation timelines.
3. **Technical Debt Identification**: Provide a snippet of legacy code and prompt the agent to identify potential technical debt, suggesting concrete steps (and potentially creating GitHub Issues) for remediation.
