## Overview
The OpenApi Specification Expert is a dedicated AI agent designed to master the entire lifecycle of API contract definition. It ensures that APIs are not only functional but also impeccably documented, adhering strictly to industry best practices like RESTful principles and OpenAPI 3.0/3.1 standards.

This agent guides you from initial high-level design concepts through to generating fully validated specifications, comprehensive documentation, and even client SDKs.

## Capabilities
*   **Specification Generation:** Creates complete API contracts in YAML or JSON format based on requirements.
*   **Validation & Quality Assurance:** Rigorously checks specs against OpenAPI standards, ensuring correct HTTP methods, status codes, and schemas.
*   **Schema Definition:** Utilizes JSON Schema for precise validation of all request and response bodies.
*   **Security Implementation:** Incorporates best practices for security schemes and authentication mechanisms directly into the spec.
*   **Documentation Output:** Generates detailed documentation, endpoint summaries, and guides for client onboarding.
*   **Automation Support:** Capable of generating sample client SDKs and automated test cases based on the final specification.

## Example Use Cases
1. **Designing a New Service:** Provide a use case (e.g., 'A user profile management service') and receive a modular, fully documented OpenAPI 3.0 YAML file.
2. **API Refinement:** Input an existing, incomplete API spec and ask the agent to review it against best practices, suggesting improvements for error handling or versioning.
3. **Client Integration Prep:** After finalizing a specification, prompt the agent to generate a sample Python client SDK stub and a comprehensive changelog guide.
