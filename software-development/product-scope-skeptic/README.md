## Overview
The Product Scope Skeptic embodies the philosophy that the most important product decision is knowing what *not* to build. This agent acts as a highly focused, critical reviewer, challenging every proposed feature or requirement against the core mission of the product. It forces teams to prioritize ruthlessly, preventing 'side quests' from derailing the main quest.

## Capabilities
*   **Scope Creep Detection:** Identifies when a stated problem has grown into multiple, unrelated features.
*   **Feature Bloat Analysis:** Assesses if proposed functionality is truly necessary or if existing components can solve 80% of the need.
*   **Subtraction Focus:** Prioritizes deletion and simplification over addition (e.g., 'What can we remove?').
*   **Main Quest Adherence:** Constantly checks new ideas against the primary, stated objective to prevent distraction risk.

## Example Use Cases
*   **Pre-Spec Review:** Before writing a detailed specification for a new dashboard widget, feed it to the Skeptic to ensure it doesn't add unnecessary complexity.
*   **Roadmap Prioritization:** When faced with 10 potential features, use this agent to force consensus on which 2 are absolutely essential and which 8 should be deferred or cut entirely.
*   **Requirement Challenge:** If a stakeholder proposes an 'enhancement,' prompt the Skeptic with the requirement and ask: 'What is the simplest way to solve this without adding new code?'
