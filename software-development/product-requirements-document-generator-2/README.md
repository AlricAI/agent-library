## Overview
This agent acts as a senior Product Manager, specializing in creating detailed and highly structured Product Requirements Documents (PRDs). It takes a high-level concept or feature request and transforms it into an actionable `prd.md` file, suitable for handing off to engineering teams.

The goal is to eliminate ambiguity by forcing the definition of user stories, acceptance criteria, technical considerations, and success metrics upfront.

## Capabilities
*   **Structured Output:** Generates PRDs using a strict Markdown format with clear headings (e.g., 'Product Goal', 'Scope', 'User Stories').
*   **Clarification Engine:** Proactively asks 3-5 targeted questions to fill in missing product details before writing the final document.
*   **Comprehensive Coverage:** Ensures all aspects, including primary flows, alternative paths, and edge cases, are documented via detailed user stories and acceptance criteria.
*   **Technical Integration:** Can analyze existing codebases (if provided) to suggest technical constraints and integration points within the PRD.

## Example Use Cases
1. **New Feature Definition:** You have an idea for a 'Dark Mode' toggle on your application and need a formal document detailing every interaction, from UI changes to backend flags.
2. **Product Specification:** You are starting a new product line and need a foundational PRD that outlines the market opportunity, target users, and core MVP features.
3. **Requirement Formalization:** A stakeholder gives you vague verbal requirements; use this agent to structure those points into testable user stories with clear acceptance criteria.
