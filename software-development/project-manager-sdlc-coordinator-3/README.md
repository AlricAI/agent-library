## Overview
This agent embodies the Project Manager (PJM) role within the MTS-SDLC-Lite framework. Its core function is to manage the 'HOW' of software delivery, focusing on execution logistics rather than product vision. It ensures that development teams stay on track by managing timelines, resources, and identifying potential blockers.

## Capabilities
*   **Timeline & Milestone Management:** Creates and maintains detailed sprint plans, setting clear milestones for deliverables.
*   **Progress Tracking:** Monitors team velocity, tracks burn-down metrics, and reports on overall project status.
*   **Coordination & Risk Management:** Coordinates dependencies between different development roles (e.g., Architect to Coder) and proactively escalates delivery risks to senior advisors.
*   **Ceremony Facilitation:** Leads key process gates such as Sprint Planning (G-Sprint) and Sprint Retrospectives (G-Sprint-Close).

## Example Use Cases
*   **Initiating a Build Phase:** When the Product Manager has finalized the 'WHAT' (the requirements), use this agent to establish the initial sprint commitment, resource allocation, and timeline.
*   **Daily Standups:** Run daily check-ins by providing status updates on tasks, identifying immediate blockers that require escalation or resolution.
*   **Pre-Release Review:** Before a major feature release, utilize its gate review capabilities to ensure all cross-functional dependencies are met and the team is ready for sign-off.
