## Overview
Scribe is a specialized AI agent designed to act as a super-gremlin expert in technical documentation and knowledge capture. Its primary function is to transform ephemeral knowledge, meeting notes, or complex processes into permanent, readable, and actionable technical artifacts.

It operates with a structured lifecycle, ensuring that every piece of work—from drafting an API reference to documenting a critical system decision—is methodical, comprehensive, and written for the benefit of future engineers.

## Capabilities
*   **Technical Documentation:** Generates clear, accurate documentation suitable for engineering teams.
*   **API Guide Creation:** Produces detailed guides for interacting with software APIs.
*   **Runbook Generation:** Creates step-by-step operational runbooks for incident response and maintenance.
*   **Decision Record Keeping (ADRs):** Captures architectural decisions, including the rationale behind them, to prevent knowledge loss.
*   **Knowledge Refreshment:** Identifies and updates stale or outdated documentation to maintain system accuracy.

## Example Use Cases
1. **Onboarding Guide Creation:** Given a complex microservice architecture, Scribe can generate a comprehensive onboarding guide that walks a new engineer through setup, core concepts, and initial tasks.
2. **Post-Incident Review:** After an outage, feed Scribe the chat logs and resolution steps; it will output a structured runbook detailing detection, mitigation, and root cause analysis.
3. **API Documentation Sync:** Provide access to raw endpoint specifications; Scribe will generate Markdown/YAML compliant documentation ready for immediate integration into developer portals.
