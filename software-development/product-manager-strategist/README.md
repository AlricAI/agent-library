## Overview
This AI agent acts as a seasoned Product Manager, specializing in strategic clarity. Its core function is to prevent engineering waste by ensuring that every proposed feature has a validated problem, a clear user segment, and measurable success criteria *before* any technical design begins. It owns the 'Why' and 'What,' keeping implementation details strictly separate.

## Capabilities
*   **Problem Framing:** Takes vague ideas and refines them into sharp, actionable problem statements.
*   **Value Hypothesis Formation:** Helps articulate testable assumptions about user needs and product value.
*   **Prioritization Frameworks:** Applies structured methods to help teams decide what to build next (and, crucially, what *not* to build).
*   **PRD Skeleton Generation:** Creates comprehensive Product Requirements Document outlines, focusing on scope, goals, and success metrics.
*   **Boundary Enforcement:** Explicitly distinguishes between strategic product decisions (owned by this agent) and technical/design implementation details (owned by other roles).

## Example Use Cases
1. **Idea Validation:** Provide a vague business goal (e.g., "Increase user engagement") and ask the agent to generate 3 distinct, prioritized value hypotheses with associated success KPIs.
2. **Scope Definition:** Feed in a list of potential features for a new dashboard and request a PRD skeleton that includes an explicit 'Out of Scope' section based on current strategic focus.
3. **Problem Refinement:** Submit raw user feedback or market research notes and ask the agent to frame the core, unmet Job-to-be-Done (JTBD) problem.
