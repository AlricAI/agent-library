## Overview
This agent acts as an expert software development assistant, taking a high-level user request (feature description, bug report, or application concept) and systematically breaking it down into three distinct, actionable markdown files: `requirements.md`, `design.md`, and `tasks.md`. It ensures that the entire project lifecycle—from initial scope definition to implementation plan—is covered in a structured, professional manner.

## Capabilities
*   **Requirements Gathering:** Creates user stories with clear acceptance criteria, covering both functional and non-functional needs.
*   **Technical Design:** Generates detailed technical specifications, including architecture diagrams (conceptually), component breakdowns, data models, error handling strategies, and testing plans.
*   **Actionable Task Breakdown:** Produces a granular implementation checklist, mapping every task back to specific requirements for traceability.
*   **Consistency Enforcement:** Maintains a consistent tone, style, and structure across all three generated documents, adhering strictly to best practices in software engineering documentation.

## Example Use Cases
1. **New Feature Implementation:** Provide a description of a new user dashboard feature; the agent will output how it should be required, designed technically, and built step-by-step.
2. **Bug Investigation:** Describe unexpected behavior (e.g., "The checkout button fails when using coupon code X"); the agent will document this as a requirement/bug fix, design potential root causes, and list debugging tasks.
3. **Application Scoping:** Request an entire application concept (e.g., "A simple inventory tracker for small businesses"); the agent will generate the full blueprint covering all necessary components and implementation phases.
