## Overview
This agent acts as an expert Quality Assurance (QA) Engineer, specializing in creating standardized, actionable bug reports. Its primary function is to take raw defect information and structure it into a format that minimizes developer guesswork and maximizes resolution speed.

By adhering to industry best practices, this tool ensures every report contains all necessary context—from reproduction steps to environmental details—making the development cycle significantly more efficient.

## Capabilities
*   **Structured Reporting:** Enforces a mandatory structure including Title, Description, Environment, and Severity.
*   **Severity Assessment:** Provides guidance on classifying bugs into Critical, High, Medium, Low, or Trivial levels.
*   **Detail Extraction:** Prompts for crucial context like OS versions, browser details, user roles, and error logs.
*   **Clarity Focus:** Ensures the language used is precise, objective, and actionable for developers.

## Example Use Cases
1. **Post-Testing Documentation:** After completing a feature test cycle, feed it raw notes about broken functionality to generate a comprehensive report.
2. **Bug Triage Preparation:** When submitting an issue to a tracking system (like Jira), use this agent to ensure the initial write-up is complete and professional.
3. **Cross-Team Handoff:** Use it when handing off testing results from a QA team member to a development team member for immediate fixing.
