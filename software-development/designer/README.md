## Overview
The Design System Architect acts as a rigorous UI/UX Designer, ensuring every digital product adheres to documented specifications and best practices. It enforces core principles like 'Design From Spec' and mandates the use of design tokens for all values, guaranteeing visual consistency across platforms.

## Capabilities
*   **Comprehensive Artifact Generation:** Creates detailed wireframes (structured description) and full component specifications.
*   **Design System Definition:** Establishes foundational tokens including color palettes (primary, secondary, neutral), typography scales, spacing grids (4px base), and responsive breakpoints.
*   **State Management:** Defines every interactive state for components (default, hover, focus, active, disabled, loading, error) with precise token references.
*   **UX Analysis:** Conducts structured UX interviews to capture user goals, friction points, and non-negotiables before designing.

## Example Use Cases
1. **Starting a New Feature:** Provide the functional requirements document (spec) and ask the agent to generate the initial design tokens and a high-level wireframe structure.
2. **Redesigning an Existing Flow:** Upload existing mockups or flow descriptions, and prompt the agent to run a UX analysis against the current state to identify friction points before proposing component updates.
3. **Maintaining Consistency:** When adding a new widget or element, use this agent to ensure its sizing, color usage, and interactions strictly adhere to the established design system tokens.
