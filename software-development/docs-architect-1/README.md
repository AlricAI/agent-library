## Overview
The Documentation Architect is a specialized AI agent designed to transform raw, complex codebases into polished, comprehensive technical documentation. It goes beyond simple docstrings by analyzing the underlying architecture, design patterns, and key decision rationales to create long-form manuals suitable for onboarding new engineers or presenting system overviews to stakeholders.

## Capabilities
*   **Codebase Analysis**: Deeply analyzes code structure, dependencies, and architectural components.
*   **System Thinking**: Maps out the 'big picture' by understanding how disparate parts interact.
*   **Structured Writing**: Organizes content into logical chapters with progressive disclosure of complexity (high-level to low-level).
*   **Decision Documentation**: Captures the *why*—the rationale behind major architectural choices, not just the *what*.
*   **Visual Planning**: Structures documentation to include detailed descriptions for necessary diagrams (e.g., sequence or flowcharts).

## Example Use Cases
1. **System Onboarding Manual**: Feed it a microservice repository and ask for an executive summary followed by component deep-dives, creating a perfect guide for new team members.
2. **Architecture Decision Record (ADR) Compilation**: Provide source code spanning multiple services and request a document detailing the major architectural trade-offs made during development.
3. **Technical Deep Dive Ebook**: Use it to structure a multi-chapter book explaining an entire domain model, ensuring consistency in terminology and flow across all sections.
