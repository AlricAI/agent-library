## Overview
This Documentation Engineer agent acts as a senior technical writer and documentation architect. Its primary function is to ensure that all project knowledge—including APIs, architecture, and usage guides—is captured in clear, maintainable, and highly usable documentation.

It moves beyond simple writing by focusing on the *system* of documentation, ensuring it scales with the codebase and remains accurate over time.

## Capabilities
*   **Documentation Gap Analysis:** Reviews existing content against best practices to pinpoint missing guides or outdated sections.
*   **API Documentation Automation:** Integrates OpenAPI/Swagger specs to generate comprehensive, interactive API reference materials.
*   **Information Architecture Design:** Structures complex knowledge bases with optimal navigation, cross-referencing, and version control integration.
*   **Tutorial & Learning Path Creation:** Designs progressive learning modules complete with hands-on code examples and progress tracking.
*   **Quality Assurance Checklist Adherence:** Ensures documentation meets technical standards like WCAG AA compliance, search optimization, and fast load times.

## Example Use Cases
1. **Onboarding New Developers:** Feed it your existing codebase structure and ask it to generate a complete 'Getting Started' guide with setup instructions, core concepts, and initial API calls.
2. **API Overhaul:** Provide OpenAPI specifications for a new microservice and request the generation of full reference docs, including error code examples and authentication walkthroughs.
3. **System Documentation Audit:** Feed it a folder containing mixed documentation types (guides, READMEs, etc.) and ask it to create a proposed information hierarchy map to improve discoverability.
