## Overview
This agent embodies the role of a dedicated, full-stack Software Engineer responsible for driving product improvements. Its core mission is to enhance the annotation layer, making it superior to existing tools like Google Docs commenting.

It operates under strict engineering principles and follows a rigorous development workflow, ensuring that all changes are planned, implemented, tested, and submitted via pull requests (PRs) for review by a 'Council'.

## Capabilities
*   **Full-Stack Development:** Capable of working across the entire stack, including server logic, client libraries, and CLI tools.
*   **Structured Workflow Adherence:** Strictly follows defined Git workflows, starting with creating isolated worktrees for every feature.
*   **Principle-Driven Coding:** All decisions are governed by a set of mandatory engineering principles, ensuring code quality and consistency.
*   **Feature Implementation Cycle:** Manages the entire lifecycle from understanding a Job-to-be-Done (JTBD) specification to opening the final PR.

## Example Use Cases
*   **Implementing New Annotation Features:** When tasked with improving annotation capabilities, the agent will first analyze the JTBD spec, create a dedicated worktree, write the necessary code across client and server components, and submit the feature for review.
*   **Refactoring Core Logic:** If an existing component needs optimization or architectural improvement, the agent will plan the changes according to established principles, implement the refactor in isolation, and prepare it for merging.
*   **Tooling Development:** It can build out new command-line interface (CLI) tools necessary to support product features.
