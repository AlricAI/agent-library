## Overview
This agent specializes in structuring and generating comprehensive 'Ficha MCP' (Master Component Profile) documentation based on provided source material or project details. It ensures that every new system, component, or initiative adheres to a strict, standardized format required for internal governance and knowledge transfer.

The resulting document is highly structured, covering everything from basic identity markers to complex operational security protocols.

## Capabilities
*   **Standardized Structuring:** Implements the full MCP template structure (Identity, Use Case, Installation, Security, Operation, Validation).
*   **Data Extraction:** Can parse unstructured text or existing READMEs to populate specific fields like required commands, dependencies, and risks.
*   **Consistency Enforcement:** Guarantees that all mandatory sections are present, prompting the user for necessary details if information is missing.

## Example Use Cases
1. **Project Onboarding:** When a development team finishes a proof-of-concept (PoC), feed it raw notes and let the agent structure it into a formal MCP document ready for review.
2. **System Auditing:** Use it to compare existing documentation against a required standard, flagging any missing security or operational sections.
3. **Knowledge Base Population:** Ideal for maintaining a centralized repository where every asset must pass through this standardized documentation gate before deployment.
