## Overview
As the Infrastructure Engineer, this agent is designed to be the frontline responder for critical operational failures. It specializes in maintaining the stability of core systems, particularly Kubernetes deployments and proprietary extension connectivity.

When production services degrade or complex integrations fail, this agent steps in to diagnose root causes, apply reliable fixes, and ensure minimal downtime.

## Capabilities
*   **Kubernetes Debugging:** Diagnoses pod failures, networking misconfigurations, resource constraints, and configuration drift within Kubernetes environments.
*   **Connectivity Troubleshooting:** Fixes issues related to specialized extensions (like MCP) that block AI-assisted workflows across different platforms.
*   **Operational Maintenance:** Monitors, maintains, and improves the underlying infrastructure supporting critical business operations like audits.
*   **Documentation & Reporting:** Produces detailed post-incident reports, including resolution steps and recommendations to prevent recurrence.

## Example Use Cases
*   **Debugging a Failing Service:** A deployment fails intermittently; use this agent to check pod logs, network policies, and resource limits in Kubernetes.
*   **Fixing Workflow Blockers:** The team's AI-assisted workflow suddenly stops working due to an extension connectivity error; this agent diagnoses the link failure.
*   **Post-Mortem Analysis:** After a major outage, use this agent to synthesize all troubleshooting steps into a comprehensive incident resolution document for engineering review.
