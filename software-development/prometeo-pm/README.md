## Overview
Prometeo acts as an elite Product Manager within a specialized 3-agent team (PM, Dev, QA). Its primary function is to define *what* needs to be built and *why*, ensuring that all product definitions are rigorous, testable, and ready for development handoff. It enforces best practices in product documentation.

## Capabilities
*   **Feature Specification Generation:** Produces comprehensive documents including problem statements, success metrics, user stories, and detailed acceptance criteria (using GIVEN/WHEN/THEN format).
*   **Scope & Risk Management:** Explicitly defines what is *in* scope and what is *out* of scope, while documenting all identified dependencies and potential risks.
*   **Process Adherence:** Follows structured methodologies like the SIGN IN / TIME OUT / SIGN OUT pause point process for clear workflow management.
*   **Handoff Preparation:** Creates a highly structured handoff package specifically tailored for a subsequent Development agent, including constraints and open questions.

## Example Use Cases
*   **New Feature Definition:** When tasked with adding a 'Dark Mode' feature, Prometeo will generate the full spec, define acceptance criteria (e.g., GIVEN user is on dark mode WHEN they navigate to settings THEN background must be black), and flag any potential conflicts.
*   **Business Requirement Analysis:** Use it to take vague business goals and transform them into prioritized, actionable product requirements with clear success metrics.
*   **Pre-Development Checkpoint:** Before handing off a project, use Prometeo to validate that all necessary documentation—including dependency mapping and open questions—is complete for the engineering team.
