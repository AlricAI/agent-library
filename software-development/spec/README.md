## Overview
This agent is designed to act as a structured Project Specification Document (PSD) generator. It enforces best practices in requirements engineering by providing a comprehensive template covering functional, non-functional, and constraint details.

It ensures that all stakeholders align on the 'what' and 'why' of a project before development begins, minimizing scope creep and ambiguity.

## Capabilities
*   **Structured Template Generation:** Creates detailed sections for Must Have (V1), Nice to Have (V2+), Non-Functional Requirements (NFRs), and Constraints.
*   **Stakeholder Alignment:** Forces the capture of rationale, owner phase, and evidence targets for every requirement.
*   **Risk Mitigation:** Includes dedicated sections for Open Questions and Client Sign-off checklists to ensure project readiness.
*   **Version Control Simulation:** Incorporates placeholders for dynamic metadata like Project Name, Date, and Version.

## Example Use Cases
1. **Initiating a New Feature:** Feed the agent a high-level concept (e.g., "Implement user profile photo upload") and ask it to generate the V1 specification draft.
2. **Scope Definition Workshop:** Use it as a guided tool during discovery sessions to ensure no critical requirement type is overlooked by the team.
3. **Handover Documentation:** Generate a final, clean spec document ready to be handed off from Product Management to the Engineering team for development sprints.
