## Overview
Project Shipper is a specialized agent designed to act as the central coordinator for any product launch, major feature release, or significant milestone. It specializes in managing the entire lifecycle from pre-launch planning through post-launch monitoring, ensuring that engineering, marketing, support, and documentation efforts are perfectly synchronized.

This agent is ideal for teams operating under tight deadlines, such as those working within a structured 6-day development cycle, where coordination failure can lead to missed market opportunities or poor user adoption.

## Capabilities
*   **Launch Orchestration:** Creates comprehensive checklists and timelines covering all necessary launch touchpoints (e.g., internal QA sign-off, marketing asset readiness).
*   **Release Calendar Management:** Structures multiple concurrent updates into a strategic, manageable release schedule to prevent feature bloat or resource contention.
*   **Go-to-Market (GTM) Strategy Development:** Helps define the narrative, target audience positioning, and key messaging required for successful market entry of new features.
*   **Post-Launch Monitoring Plan:** Establishes critical monitoring phases immediately following a release to track initial adoption rates and identify urgent hotfixes or improvements.

## Example Use Cases
*   **Major Feature Rollout:** When announcing a significant new capability, use Project Shipper to ensure marketing assets are ready *before* the code is deployed, preventing PR gaps.
*   **Multi-Sprint Coordination:** If three distinct updates are scheduled for one sprint, this agent will help sequence them logically—releasing foundational features first and iterative improvements later.
*   **Market Positioning:** Before launching a feature, prompt it to develop several GTM angles (e.g., 'Productivity Booster' vs. 'Collaboration Enhancer') based on the intended user persona.
