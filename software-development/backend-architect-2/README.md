## Overview
This agent acts as an expert backend system architect, specializing in designing highly scalable, resilient, and maintainable distributed systems. Its core philosophy centers on establishing clear service boundaries, defining robust contracts upfront, and baking resilience patterns into the architecture from day one.

## Capabilities
* **Comprehensive API Design:** Masters RESTful APIs (including versioning and pagination), GraphQL schemas, gRPC services using Protocol Buffers, and real-time WebSocket/SSE implementations.
* **Architectural Patterns:** Proficient in designing event-driven architectures, implementing service mesh patterns, and defining clear inter-service communication strategies.
* **Contract Definition:** Expertise in API-First design principles, generating OpenAPI/Swagger specifications, and managing complex data contracts.
* **Resilience & Observability:** Incorporates best practices for failure handling, circuit breakers, retry logic, and comprehensive observability hooks into the initial design.

## Example Use Cases
* **Designing a New E-commerce Backend:** Requesting an architecture that separates Product Catalog, Order Processing, and Inventory services using Kafka/RabbitMQ for asynchronous communication.
* **API Specification Generation:** Providing requirements for a new client feature (e.g., user profile updates) and receiving a complete OpenAPI 3.0 specification draft.
* **Technology Stack Review:** Asking to compare the trade-offs between implementing a service boundary using GraphQL Federation versus traditional REST endpoints.
