## Overview
The Developer Experience (DX) Optimizer acts as a dedicated specialist focused on eliminating friction from the software development lifecycle. Its goal is to make development joyful, productive, and seamless by optimizing everything from initial setup to daily coding routines.

## Capabilities
*   **Environment Setup:** Streamlines onboarding processes to take minutes, automating dependency management and providing intelligent defaults.
*   **Workflow Automation:** Identifies repetitive tasks suitable for automation, creating useful aliases, shortcuts, and optimizing build/test cycles.
*   **Tooling Enhancement:** Configures IDE settings, sets up necessary git hooks, and generates project-specific CLI commands.
*   **Documentation Generation:** Creates functional setup guides with interactive examples and maintains accurate troubleshooting documentation.

## Example Use Cases
1. **New Project Onboarding:** Run this agent when starting a greenfield repository to instantly generate optimized `package.json` scripts, necessary `.gitignore` rules, and a comprehensive README that guides new team members through the first run.
2. **Performance Bottleneck Identification:** If testing reveals slow build times or complex setup steps, use the agent to profile the current workflow, suggest best-practice tooling replacements, and implement incremental improvements for measurable speed gains.
3. **Team Feedback Integration:** After receiving feedback like, "It takes too long to get this running," deploy the DX Optimizer to analyze the existing pain points and deliver concrete fixes, such as adding a `Makefile` or refining CI/CD scripts.
