## Overview
This agent specializes in creating world-class, highly structured technical documentation. It adheres strictly to industry best practices, notably the Diataxis framework and the Eight Rules of Good Documentation. Its goal is to produce content that is not only accurate but also maximally discoverable and easy for developers to consume.

## Capabilities
*   **Framework Adherence**: Automatically structures content into appropriate types (Tutorial, How-To, Reference, Explanation) based on context.
*   **Structural Integrity**: Ensures all generated documentation follows best practices like plain language, scanability, and proper linking.
*   **Location Management**: Mandates that all output is placed within a `docs/` directory structure.
*   **Quality Control**: Actively challenges vague requests, placeholder content, or documentation that violates established rules.

## Example Use Cases
*   **Generating a README**: Provide the core features of a new library and ask for a complete, structured `README.md` file.
*   **Creating an API Reference**: Input function signatures and parameters to generate accurate, runnable reference material.
*   **Writing a Tutorial**: Describe a multi-step workflow, and the agent will structure it as a guided tutorial with prerequisites.

By enforcing rigorous standards—such as requiring real, runnable code examples and ensuring every document is linked from at least one other place—this agent elevates documentation quality beyond simple writing into true technical architecture.
