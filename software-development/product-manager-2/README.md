## Overview
This agent acts as a seasoned Product Manager, taking vague or high-level product requests and structuring them into formal, actionable Product Requirement Documents (PRDs). It ensures that every requirement is backed by clear acceptance criteria and that non-functional aspects are considered from the outset.

## Capabilities
*   **Structured Documentation:** Generates PRDs following industry best practices, including context setting and success metrics.
*   **Requirement Detailing:** Formats functional requirements with explicit, testable acceptance criteria for engineering teams.
*   **Non-Functional Requirements (NFRs):** Automatically incorporates critical considerations like performance, security, scalability, and SLOs/SLAs.
*   **Scope Management:** Clearly defines what is in scope and, critically, what is out of scope to prevent scope creep.
*   **Risk Assessment:** Includes sections for rollout plans, identified risks, and open questions for stakeholders to address.

## Example Use Cases
1. **Feature Definition:** Input a request like, "We need to improve user onboarding," and receive a PRD detailing specific steps, success metrics (e.g., 20% reduction in drop-off rate), and necessary technical requirements.
2. **Product Vetting:** When evaluating a new market opportunity, use this agent to structure the initial feasibility document, ensuring all stakeholders agree on measurable goals before writing any code.
3. **Stakeholder Alignment:** Use it to create a single source of truth for a complex feature rollout, forcing clarity on technical constraints and business acceptance criteria.
