## Overview
This agent simulates the role of a VP of Product for 'Olivia,' a local-first household command center. Its core function is to own and drive product strategy, ensuring that all feature development aligns with the overall product vision while maintaining high documentation quality.

It acts as the central point for translating ambiguous stakeholder requests into clear, executable specifications, managing cross-team dependencies between design and engineering.

## Capabilities
*   **Product Strategy:** Maintains alignment between business goals and product direction.
*   **Feature Specification:** Drafts comprehensive feature specs using a defined template, detailing workflow, system behavior, trust models, and acceptance criteria.
*   **Documentation Management:** Keeps critical documents like the roadmap, decision history, and learnings up-to-date.
*   **Coordination & Unblocking:** Resolves ambiguities in specifications to unblock designers and engineers.
*   **Release Readiness:** Enforces mandatory release checks by reviewing Git logs before any task execution.

## Example Use Cases
*   **New Feature Definition:** When a new household feature is requested, use this agent to draft the full specification document, ensuring all acceptance criteria are met before handing it off for design and implementation.
*   **Roadmap Gap Analysis:** Reviewing the current milestones against product vision documents to recommend necessary pivots or prioritization changes.
*   **Changelog Generation:** Drafting user-facing entries for version bumps based on completed features, maintaining a consistent voice across releases.
