## Overview
This Prompt Engineer acts as the dedicated AI expert for optimizing the underlying instructions (prompts) of all other specialized agents within the VorstersNV ecosystem. Its core function is to move beyond simple fixes by implementing structured, iterative prompt engineering cycles based on performance metrics and best practices.

It guides users through a formal process involving measurement, diagnosis, hypothesis generation, writing, testing, and comparison to ensure continuous quality improvement for critical business functions like customer service or order processing.

## Capabilities
*   **Systematic Iteration:** Implements a defined 7-step cycle (Measure $\rightarrow$ Diagnose $\rightarrow$ Hypothesize $\rightarrow$ Write $\rightarrow$ Test $\rightarrow$ Compare $\rightarrow$ Commit) for prompt refinement.
*   **Technique Application:** Expertly applies advanced prompting patterns, including Chain-of-Thought (CoT) for analytical agents and Few-Shot Examples for content generation agents.
*   **Agent Mapping:** Maintains awareness of various agent components (e.g., `klantenservice`, `product_beschrijving`) and their associated prompt locations.

## Example Use Cases
*   **Improving Customer Service:** If the 'Customer Service' agent provides vague answers, invoke this agent to analyze its current system prompt against recent negative feedback scores.
*   **Enhancing Content Quality:** When a product description agent needs more persuasive copy, use this agent to inject Few-Shot Examples into its pre-prompt template.
*   **General Optimization:** Use it when any agent's performance degrades or when you suspect the underlying instructions are suboptimal, prompting for an 'agent prompt verbeteren' cycle.
