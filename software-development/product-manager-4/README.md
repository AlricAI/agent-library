## Overview
This agent is designed to act as a seasoned Product Manager, taking vague or high-level feature requests and structuring them into professional, actionable Product Requirement Documents (PRDs). It ensures that all necessary components—from user context to non-functional requirements—are covered, making the output immediately usable by engineering teams.

## Capabilities
*   **Structured Output:** Generates PRDs following a strict, industry-standard format.
*   **Contextual Framing:** Automatically includes sections for 'Context & Why Now,' defining user needs and business justification.
*   **Detailed Requirements:** Formulates numbered functional requirements, each paired with precise acceptance criteria.
*   **Non-Functional Coverage:** Incorporates critical NFRs such as performance, security, scalability, and observability.
*   **Scope Definition:** Clearly delineates what is 'in scope' versus what is explicitly 'out of scope,' along with risk assessments and rollout plans.

## Example Use Cases
1. **New Feature Specification:** Provide a concept like, "We need to improve user onboarding," and receive a PRD detailing required flows, acceptance criteria for each step, and necessary performance metrics (e.g., time-to-completion).
2. **Product V1 Definition:** When launching a new product line, use this agent to structure the entire initial vision into a comprehensive document that guides design and development.
3. **Stakeholder Alignment:** Use it to consolidate feedback from multiple departments into one single source of truth for engineering kickoff meetings.
