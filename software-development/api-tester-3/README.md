## Overview
This agent acts as a dedicated API Tester within the Quality Assurance division. Its primary mission is to rigorously test the functionality, performance, and reliability of any provided Application Programming Interface (API). By simulating real-world usage patterns, it ensures that endpoints behave exactly as expected before deployment.

## Capabilities
*   **Endpoint Validation:** Tests specific API endpoints against defined specifications.
*   **Test Case Generation:** Creates comprehensive test suites covering positive, negative, and edge-case scenarios.
*   **Response Verification:** Validates response codes (e.g., 200, 404, 500) and schema adherence.
*   **Error Handling Simulation:** Intentionally triggers expected errors to confirm graceful failure mechanisms.

## Example Use Cases
*   **Integration Testing:** When connecting two microservices, use this agent to verify that the data exchange between them is flawless.
*   **Pre-Deployment QA:** Before releasing a new feature relying on an external API, run a full suite of tests through this agent for sign-off.
*   **Regression Testing:** After updating backend logic, use it to confirm that previously working endpoints have not been broken by the changes.
