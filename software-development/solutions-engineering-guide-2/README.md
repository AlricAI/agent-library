## Overview
This agent is designed for the critical 'middle lane' of the sales cycle where technical credibility must be established. It moves beyond simple qualification by helping teams create concrete, actionable evaluation plans that map directly to a buyer's existing technology stack and workflow.

Its core function is to ensure all technical discussions are grounded in current product truth, clearly defining what is supported, what requires integration, and what constitutes a true product gap versus a solvable education problem.

## Capabilities
*   **Site Evaluation Planning:** Generates detailed evaluation plans that map specific buyer workflows to existing platform features.
*   **Boundary Definition:** Clearly articulates technical boundaries regarding hosted access, data exports, packaging limitations, and deployment readiness.
*   **Actionable Next Steps:** Converts vague discussions into short, executable checklists for both the buyer and the internal team.
*   **Gap Identification:** Systematically surfaces product gaps as explicit work items rather than allowing them to remain ambiguous in conversation.

## Example Use Cases
*   **Evaluating a Complex Integration:** A prospect needs to know if their legacy CRM can push data through a specific workflow. This agent generates a checklist detailing required APIs, necessary middleware steps, and current product limitations for that flow.
*   **Differentiating Gaps vs. Education:** When a buyer asks for a feature, the agent helps the team determine if it's a known limitation requiring roadmap discussion or if it’s simply a misunderstanding of existing functionality.
*   **Pre-Deployment Readiness Check:** Before signing off on a proof-of-concept, this agent forces the creation of a 'Readiness Artifact Checklist,' ensuring all deployment prerequisites (security sign-off, data mapping, etc.) are documented and owned.
