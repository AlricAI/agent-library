## Overview
The Solutions Engineering Guide agent is designed to occupy the critical 'middle lane' of the buyer journey—the space between initial qualification and final commitment. Its core purpose is to establish technical credibility by providing concrete, truthful evaluations of complex site packages against a buyer's existing technology stack and workflow.

It ensures that discussions move beyond vague assurances by focusing on actionable technical boundaries, deployment readiness, and clear next steps.

## Capabilities
*   **Site Evaluation Planning:** Creates detailed evaluation plans mapped directly to the buyer’s actual technology stack and operational workflows.
*   **Boundary Definition:** Clearly defines technical limitations regarding hosted sessions, data exports, packaging requirements, and deployment status.
*   **Actionable Next Steps:** Replaces ambiguous statements with concrete, executable checklists for both the buyer and the internal team.
*   **Gap Identification:** Surfaces product gaps as explicit, documented work items rather than letting them remain hidden in conversation.

## Example Use Cases
1. **Evaluating a Custom Integration:** A prospect asks how their legacy CRM will connect to the new platform. The agent generates a checklist detailing required APIs, necessary middleware steps, and current integration limitations.
2. **Clarifying Deployment Scope:** When discussing moving from staging to production, the agent forces a review of existing artifacts, ensuring no deployment step is assumed without documented evidence.
3. **Differentiating Needs vs. Gaps:** If a buyer asks for a feature that doesn't exist, the agent helps distinguish if it’s a simple education gap (a process change) or a true product capability gap requiring engineering effort.
