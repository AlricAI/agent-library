## Overview
The System Architect Planner (SOUL) is designed for technical leads and product managers who need to translate abstract concepts into concrete, executable development roadmaps. It emphasizes pragmatism over theoretical perfection, ensuring that every output artifact can be immediately picked up by a builder with minimal ambiguity.

This agent avoids the pitfalls of overly vague or excessively decomposed plans, focusing instead on delivering structured milestones suitable for iterative delivery cycles.

## Capabilities
*   **Pragmatic Scoping:** Narrows down large ideas into manageable, deliverable chunks.
*   **Structured Output:** Ensures all plans are deterministic and follow clear, actionable formats.
*   **Builder Focus:** Outputs are written with the end-user developer in mind, minimizing assumptions.
*   **Iterative Planning:** Structures work into distinct phases that allow for continuous integration and feedback.

## Example Use Cases
*   **From Idea to MVP:** Provide a core product idea (e.g., "A microservice tracking user sentiment from tweets") and receive a phased plan detailing the necessary components, APIs, and initial endpoints needed for a Minimum Viable Product (MVP).
*   **Refining Complex Features:** If a feature description is too broad ("Improve user engagement"), feed it to SOUL and ask for a structured breakdown into 3-5 measurable, buildable sprints.
*   **Technical Debt Reduction Plan:** Input a list of identified technical shortcomings and request a prioritized, actionable plan with estimated effort levels for remediation.
