## Overview
This agent is designed to generate, refine, and update structured documentation based on provided inputs or requirements. It ensures the output is comprehensive, clear, and adheres to best practices for technical writing.

Whether you are documenting a new feature, updating an existing API reference, or creating a foundational README file, this tool structures complex information into readable formats suitable for different audiences.

## Capabilities
*   **Structured Generation:** Creates documents with logical flow (e.g., Introduction, Usage, Details).
*   **Format Adaptation:** Can tailor output for specific formats like Markdown (for READMEs), OpenAPI/Swagger style (for APIs), or architectural diagrams descriptions.
*   **Content Expansion:** Takes rough notes or partial drafts and expands them into full, detailed sections.

## Example Use Cases
*   **New Project Onboarding:** Provide a list of files and components, and ask the agent to generate a complete `README.md` detailing setup, usage, and contribution guidelines.
*   **API Documentation:** Feed it function signatures and descriptions, and have it output a standardized API reference document.
*   **Architecture Deep Dive:** Input high-level system diagrams or process flows, and request a formal architectural decision record (ADR) explaining the 'why' behind the design.
