## Overview
This agent acts as a dedicated API Documentation Specialist, ensuring that your APIs are not just functional but also exceptionally well-documented for developers. It moves beyond simple endpoint listings to create comprehensive, developer-centric documentation packages.

When you provide an API definition or endpoints, this tool will generate industry-standard artifacts, including OpenAPI 3.0 specifications, client SDKs in multiple languages, and interactive testing collections.

## Capabilities
*   **OpenAPI Specification Generation:** Creates complete OpenAPI 3.0/Swagger files with detailed endpoint descriptions, request/response schemas, and validation rules.
*   **SDK & Client Library Creation:** Generates usable client libraries for various programming languages (e.g., Python, JavaScript) based on the API contract.
*   **Interactive Documentation:** Builds documentation that includes runnable examples, often compatible with tools like Swagger UI or Postman collections.
*   **Developer Experience Focus:** Prioritizes actionable content, including `curl` examples, common integration patterns, and detailed authentication guides.
*   **Versioning & Error Handling:** Designs robust versioning strategies and comprehensive error code reference sections with clear resolution steps.

## Example Use Cases
1. **New API Launch:** Provide a set of endpoints, and the agent will return a full OpenAPI spec, initial SDK stubs, and an authentication guide ready for internal review.
2. **API Refactoring/Versioning:** If you change an endpoint's payload, use this agent to generate both the updated OpenAPI schema *and* a migration guide detailing how existing clients should update their code.
3. **Onboarding New Developers:** Feed it your existing API documentation gaps; it will structure and populate missing sections like error handling codes or language-specific usage examples.
