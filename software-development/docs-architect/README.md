## Overview
The Documentation Architect is a specialized AI designed to transform raw, complex codebases into polished, comprehensive technical documentation. It goes beyond simple README generation by understanding the 'why' behind the code—the architectural decisions and design patterns—and structuring this knowledge for maximum clarity.

This agent excels at creating long-form manuals suitable for onboarding new engineers, documenting system boundaries for stakeholders, or serving as a definitive reference guide for complex software systems.

## Capabilities
*   **Codebase Analysis**: Deeply analyzes code structure, dependencies, and underlying architectural patterns.
*   **Structured Documentation**: Creates logical, navigable documentation hierarchies with clear chapter divisions.
*   **Progressive Disclosure**: Structures content to move logically from high-level overviews (for executives) down to granular implementation details (for engineers).
*   **Rationale Capture**: Explicitly documents the *design decisions*, providing context and justification for architectural choices.
*   **Visual Aid Description**: Generates detailed descriptions of necessary diagrams, such as sequence or flowcharts, to complement the written text.

## Example Use Cases
1. **System Onboarding Guide**: Feed it a new microservice repository and ask it to generate a 50-page guide covering setup, core components, data flows, and API usage for new team members.
2. **Architecture Deep Dive**: Provide the entire codebase and request an 'Architectural Decision Record' (ADR) style document detailing why certain patterns were chosen over alternatives.
3. **Technical Book Creation**: Use it to structure a book chapter on a complex domain, ensuring all necessary sections—Executive Summary, Core Components, Design Decisions—are present and cross-referenced.
