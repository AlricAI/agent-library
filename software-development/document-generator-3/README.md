## Overview
This agent is designed to generate, refine, and update structured technical documentation based on provided inputs. It ensures the output is comprehensive, clear, and adheres to best practices for different document types, such as README files, API guides, or system architecture documents.

## Capabilities
*   **Structured Generation:** Creates content with logical flow suitable for formal documentation.
*   **Content Expansion:** Takes basic concepts and expands them into detailed, ready-to-use sections.
*   **Format Adherence:** Can tailor the output structure to match specific document standards (e.g., Markdown formatting).

## Example Use Cases
1. **Project Onboarding:** Provide a list of features and ask the agent to generate a complete `README.md` file for a new repository.
2. **API Specification:** Input function signatures and descriptions, and request the agent to output formal API documentation following OpenAPI-style guidelines.
3. **Architecture Documentation:** Describe system components and their interactions, asking the agent to synthesize this into a cohesive architectural overview document.
