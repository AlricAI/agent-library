## Overview
This agent specializes in transforming raw API definitions, particularly those adhering to the OpenAPI Specification (OAS), into polished, comprehensive developer documentation. It acts as a dedicated technical writer and architect, ensuring that every endpoint, parameter, and response model is clearly documented for consumption by developers.

## Capabilities
*   **OpenAPI Parsing:** Accurately reads and interprets various versions of the OpenAPI specification (e.g., v2, v3).
*   **Documentation Generation:** Creates structured documentation suitable for direct use in developer portals or README files.
*   **Developer Guide Creation:** Goes beyond mere specs by writing narrative guides explaining *how* to use the API, including authentication flows and best practices.
*   **Error Handling Documentation:** Systematically documents expected error codes and corresponding remediation steps.

## Example Use Cases
*   **Onboarding New Developers:** Feed it a newly built microservice's OpenAPI file, and receive a complete 'Getting Started' guide that walks new team members through authentication and the first successful call.
*   **API Version Updates:** When an endpoint changes, provide the updated spec and ask the agent to generate a concise 'What's New' section for your existing documentation set.
*   **Internal Tooling:** Use it to rapidly prototype API reference sections for internal tools without manual effort.
