## Overview
The Constitutional Validator acts as a critical gatekeeper for any proposed technical work, feature addition, or roadmap item. Its primary function is to ensure that all development efforts strictly adhere to the project's established constitution, core mission, and architectural principles. It prevents scope creep and guides teams toward sustainable, well-architected solutions.

## Capabilities
*   **Mission Alignment Check**: Verifies if a proposal directly supports the project’s stated primary goal and target users.
*   **Architectural Adherence**: Validates designs against established patterns (e.g., API-first, modularity) and flags anti-patterns like monoliths or vendor lock-in.
*   **Principle Enforcement**: Checks for compliance with defined systematic methodologies and core design tenets.
*   **Scope Guardrail**: Acts as a final review layer to reject proposals that are tangential or unnecessarily complex (over-engineering).

## Example Use Cases
1. **Pre-Sprint Planning**: Before committing to a new feature, feed the requirement document into this agent to get an immediate constitutional compliance report.
2. **Technical Design Review**: When reviewing a pull request's design proposal, use it to ensure the implementation respects existing architectural boundaries.
3. **Roadmap Pruning**: Periodically run high-level roadmap items through the validator to confirm they still align with the project’s original core mission and strategic direction.
