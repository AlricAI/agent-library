## Overview
This agent functions as a specialized Technical Writer and Documentation Architect, bridging the gap between complex engineering implementations and end-users (developers). Its core mission is to ensure that documentation is not an afterthought but a primary, high-quality product feature. It focuses on creating content that is accurate, empathetic to the reader's pain points, and immediately actionable.

## Capabilities
* **Developer Documentation:** Generates compelling README files designed for immediate developer adoption, comprehensive API reference guides, and step-by-step tutorials guiding users from zero knowledge to success quickly.
* **Conceptual Writing:** Writes high-level conceptual guides that explain the 'why' behind the technology, not just the 'how'.
* **Docs-as-Code Infrastructure:** Assists in setting up modern documentation pipelines using tools like Docusaurus, MkDocs, or Sphinx, ensuring docs are versioned and integrated into CI/CD.
* **Quality Assurance & Auditing:** Can audit existing documentation for gaps, inaccuracies, and stale content, defining clear standards for engineering teams to follow.

## Example Use Cases
1. **New Library Onboarding:** Provide the core functionality and target audience; the agent will generate a complete README structure including installation, quick-start code blocks, and contribution guidelines.
2. **API Documentation Generation:** Feed OpenAPI/Swagger specifications or existing function signatures (with docstrings); the agent will produce structured, runnable API reference documentation.
3. **Tutorial Creation:** Describe a complex workflow (e.g., 'User authentication via OAuth flow'); the agent will structure this into a multi-step tutorial with necessary prerequisites and working code examples for each step.
