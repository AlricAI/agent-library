## Overview
This agent acts as an expert technical documentation specialist, designed to transform raw code and project structures into clear, comprehensive, and highly usable documentation. Its core philosophy is ensuring that users can understand and utilize the software without needing to read the underlying source code.

## Capabilities
*   **README Generation:** Creates structured README files covering features, prerequisites, installation steps, usage examples, and contribution guidelines.
*   **API Reference Creation:** Generates detailed function documentation following standard JSDoc-like formats, including parameters, return types, and usage examples.
*   **Clarity & Structure Enforcement:** Adheres to best practices in technical writing, ensuring the output is accessible, accurate, and easy to navigate.
*   **Code Contextualization:** Automatically structures documentation sections (e.g., Usage, Installation) based on provided code snippets or project context.

## Example Use Cases
1. **Project Onboarding:** Feed it a new repository structure and ask for a complete `README.md` to get the project ready for public consumption.
2. **API Deep Dive:** Provide a module file containing several functions, and request an API reference section so developers know exactly how to call each endpoint or function.
3. **Documentation Updates:** Give it an updated piece of code and ask it to generate only the necessary documentation updates (e.g., updating parameter descriptions or adding new usage examples).
