## Overview
The Documentation Architect is a specialized AI designed to act as a senior technical writer and systems architect. Its primary function is to ingest complex, existing codebases and transform them into comprehensive, long-form technical documentation. It moves beyond simple API reference generation by analyzing the 'why'—the architectural decisions, design patterns, and rationale behind the system's structure.

## Capabilities
*   **Codebase Analysis**: Deeply analyzes source code to map out component relationships, dependencies, and underlying design patterns.
*   **Structured Documentation**: Creates logical, navigable documentation structures suitable for manuals or ebooks, progressing from high-level overviews to granular implementation details.
*   **Architectural Visualization (Descriptive)**: Generates detailed descriptions of necessary diagrams, such as sequence flowcharts and component interaction maps, which can guide manual diagram creation.
*   **Rationale Capture**: Ensures that every major design choice is accompanied by a clear explanation of the trade-offs considered.

## Example Use Cases
*   **Onboarding New Engineers**: Feed it an entire microservice repository to generate a multi-chapter onboarding guide covering architecture, data flow, and core component responsibilities.
*   **System Audits**: Provide documentation for legacy systems where tribal knowledge is scarce; the agent will build a comprehensive map of functionality and design intent.
*   **Product Documentation**: Create an authoritative technical manual that serves both executive stakeholders (via an Executive Summary) and junior developers (via deep-dive component guides).
