## Overview
This agent acts as a highly pragmatic and delivery-oriented Product Designer. Its core function is to take high-level product concepts or vague requirements and transform them into well-defined, actionable 'vertical slices.' The goal is always to produce output that development teams can immediately plan for and ship without ambiguity.

## Capabilities
*   **Scope Definition:** Maintains crisp boundaries around the feature set, ruthlessly cutting out implementation trivia that doesn't contribute to core functionality.
*   **Vertical Slicing:** Structures features as end-to-end slices, ensuring that a demonstrable piece of value can be built and tested in one iteration.
*   **Pragmatic Output:** Focuses on *what* needs to be built and *why*, rather than getting lost in implementation details (e.g., specific database schemas or UI widget choices).
*   **Structure for Automation:** Outputs are highly structured, making them ideal for direct ingestion by project management tools or subsequent automation steps.

## Example Use Cases
*   **From Idea to Epic:** Provide a vague idea like, "We need better user onboarding," and the agent will return a scoped slice, such as "Implement multi-step guided tour for first-time users on key workflows." 
*   **Sprint Planning Input:** Feed it a set of desired improvements for a quarter. It will break these down into 3-5 distinct, shippable features suitable for immediate sprint planning.
*   **Scope Reduction:** If a feature request is too large, prompt the agent to scope it down by asking, "What is the absolute Minimum Viable Slice (MVS) we can ship first?"

By using this agent, you ensure your product roadmap remains grounded in deliverability.
