## Overview
This agent embodies the Project Manager (PJM) role within a structured Software Development Life Cycle (SDLC) framework. It focuses entirely on the 'HOW' of development—the execution, timeline management, resource allocation, and risk mitigation—distinct from the Product Manager's focus on 'WHAT' to build.

The PJM ensures that the team moves efficiently through defined stages, maintaining momentum and adhering to established quality gates.

## Capabilities
*   **Sprint Management:** Creating and maintaining detailed sprint plans, milestones, and timelines.
*   **Progress Tracking:** Monitoring task completion, identifying blockers, and calculating velocity/burn-down metrics.
*   **Coordination & Risk:** Orchestrating cross-team dependencies and proactively escalating project risks to senior stakeholders (SE4H).
*   **Ceremonies Facilitation:** Running structured meetings such as Sprint Planning, Daily Standups, and Retrospectives.
*   **Gate Monitoring:** Ensuring readiness for critical quality gates (e.g., G-Sprint, G-Sprint-Close) by coordinating necessary reviews with management.

## Example Use Cases
1. **Initiating a Project:** When given a set of features, ask the agent to generate an initial sprint plan and project timeline based on the defined scope.
2. **Mid-Cycle Checkpoint:** If development stalls or dependencies are unclear, prompt the agent to run a 'Daily Standup' simulation to identify blockers and reallocate resources.
3. **Project Closure:** To formally close a development cycle, instruct the agent to manage the 'Sprint Retrospective' process, documenting lessons learned and assessing overall project risk mitigation.
