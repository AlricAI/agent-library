## Overview
The Backend Architect is a specialized consultative agent designed to guide the planning and design of complex, enterprise-grade backend systems. It operates by first deeply understanding project requirements through structured questioning before proposing detailed, multi-faceted architectural blueprints.

This agent ensures that all proposed solutions adhere to modern software engineering best practices, prioritizing scalability, maintainability, and security from the initial concept phase.

## Capabilities
*   **System Design:** Proposes appropriate architectures (e.g., microservices, event-driven) with clear service boundaries.
*   **API Specification:** Designs robust APIs using RESTful principles, GraphQL schemas, or gRPC services, including versioning strategies.
*   **Data Modeling:** Recommends optimal database selections and detailed schema designs, incorporating indexing and caching layers.
*   **Scalability Planning:** Outlines strategies for horizontal scaling, load balancing, and performance optimization to handle growth.
*   **Security Integration:** Embeds security patterns (AuthN/AuthZ) directly into the architecture design process.

## Example Use Cases
*   **Designing a Payment Gateway:** You can task it with architecting a new payment processing module, ensuring compliance, transaction integrity, and high availability across multiple services.
*   **Migrating a Monolith:** Use it to analyze an existing monolithic application and propose a phased, risk-mitigated migration path to a microservices architecture.
*   **Implementing Real-Time Features:** Ask for the design of a real-time notification system, requiring expertise in message queues (like Kafka) and WebSocket integration.
