## Overview
This agent simulates the role of a Chief Executive Officer (CEO) for a software project, ensuring that all development outcomes meet rigorous quality standards. Its primary function is not to write code but to own the entire outcome lifecycle: from initial issue triage through specialist delegation and final verification.

The CEO enforces strict 'Definition of Done' criteria, meaning no work is considered complete until every step—including acceptance criteria definition, correct specialist assignment, verified build passes, and confirmation against gold-star metrics—is met.

## Capabilities
*   **Issue Triage:** Systematically analyzes incoming issues to determine severity, domain, and necessary files for resolution.
*   **Acceptance Criteria Definition:** Mandates the creation of measurable acceptance criteria before any work begins, preventing ambiguous development.
*   **Delegation & Staffing:** Assigns tasks only to the appropriate specialist agent with clear rationale.
*   **Verification & Review:** Verifies deliverables by checking build passes and comparing simulator screenshots against defined 'Gold Star' expectations.
*   **Feedback Loop Management:** Provides specific, actionable feedback on failed deliveries rather than vague requests to retry.

## Example Use Cases
*   **Complex Feature Implementation:** When a large feature requires multiple specialized components (e.g., UI, Backend Logic, Safety Checks), use this agent to manage the workflow sequentially and ensure integration testing passes.
*   **Quality Gatekeeping:** If development stalls due to unclear requirements or substandard work, deploy this agent to enforce the necessary checkpoints before proceeding.
*   **Project Status Reporting:** To get a definitive status update on an issue, ensuring it hasn't just been 'worked on,' but is fully verified and closed out according to established protocols.
