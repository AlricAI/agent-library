## Overview
The Frontend Design Architect agent is specialized in bridging the gap between visual design and functional code. It takes abstract concepts, static mockups (like Figma files or screenshots), or low-fidelity wireframes and transforms them into comprehensive, developer-ready technical specifications.

This agent ensures that the final implementation will be pixel-perfect while adhering to scalable architectural patterns, making it invaluable for maintaining design consistency across large applications.

## Capabilities
*   **Design Analysis:** Interprets UI/UX designs from various inputs (mockups, screenshots) to understand layout, components, and user flows.
*   **Technical Specification Generation:** Creates detailed documentation outlining required technologies, component props, state management needs, and interaction logic.
*   **Component Architecture:** Decomposes complex screens into reusable, atomic components following best practices for modern frontend frameworks (e.g., React).
*   **Design System Creation:** Extracts visual tokens, patterns, and guidelines from existing UI elements to build a cohesive design system specification.

## Example Use Cases
*   **Mockup Implementation:** If you provide a Figma mockup of a complex dashboard, this agent will generate the necessary component breakdown (e.g., `Card`, `Sidebar`, `DataTable`) along with suggested state management hooks needed for implementation in React or Vue.
*   **Design System Auditing:** Uploading several screenshots from different parts of an application allows the agent to identify inconsistencies and propose a unified, scalable design system structure.
*   **Wireframe Structuring:** For a hand-drawn or basic wireframe representing a user profile page, it will output a hierarchical component tree, detailing which components need to be built first and how they should interact.
