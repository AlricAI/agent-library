## Overview
The Tdd Orchestrator is an elite AI agent designed to enforce rigorous and disciplined Test-Driven Development (TDD) practices across your entire software development lifecycle. It masters the complete red-green-refactor cycle, moving beyond simple testing to actively orchestrate multi-agent workflows that ensure comprehensive test coverage while maintaining high development velocity.

## Capabilities
*   **Red-Green-Refactor Cycle Enforcement:** Manages and enforces the core TDD rhythm, ensuring developers write failing tests first, then make them pass, and finally refactor safely.
*   **Multi-Agent Coordination:** Orchestrates specialized testing agents (unit, integration, E2E) to run coordinated test suites across multiple development streams.
*   **Methodology Integration:** Supports various modern practices including Acceptance Test-Driven Development (ATDD), Behavior-Driven Development (BDD), and classic TDD approaches.
*   **Compliance & Governance:** Actively detects and prevents TDD anti-patterns, providing automated compliance checks for team adherence to best practices.
*   **Productivity Optimization:** Measures cycle time and optimizes the development flow state to provide rapid feedback loops.

## Example Use Cases
1. **New Feature Implementation:** When starting a new feature, instruct the agent to guide the process: first generate failing acceptance tests (Red), then write minimal code to pass them (Green), and finally suggest safe refactoring passes (Refactor).
2. **Cross-Team Sync:** If multiple teams are working on interconnected modules, use this agent to synchronize their respective TDD efforts, ensuring consistent testing standards across repositories.
3. **Code Review Governance:** Integrate it into your CI/CD pipeline to automatically validate that all new pull requests have passed through a documented TDD governance checkpoint before merging.
