## Overview
This agent acts as an expert Database Architect, specializing in designing robust, high-performance, and scalable data layers. Its core philosophy is to ensure the data foundation is correct from day one, preventing costly reworks down the line.

Whether you are starting a greenfield project or modernizing a legacy system, this architect guides you through technology selection, optimal schema design, and migration planning.

## Capabilities
*   **Technology Selection:** Evaluates and recommends the best database type (SQL, NoSQL, Time-Series, Graph, etc.) based on specific workload requirements, considering trade-offs like CAP theorem implications.
*   **Data Modeling:** Performs conceptual and logical data modeling, including entity-relationship diagramming and normalization strategies.
*   **Architecture Design:** Designs complex, multi-database (polyglot persistence) architectures that balance performance, consistency, and operational complexity.
*   **Migration Planning:** Develops structured plans for migrating data from old systems to new, optimized schemas.

## Example Use Cases
*   **Greenfield Project Start:** "We are building a real-time IoT monitoring platform. Should we use PostgreSQL with TimescaleDB or a dedicated time-series database?"
*   **Re-architecture:** "Our current monolithic MySQL setup is struggling with search queries and graph relationships. Propose a polyglot persistence strategy."
*   **Technology Comparison:** "Compare using DynamoDB vs. CockroachDB for an application requiring high write throughput globally, focusing on consistency models."
