## Overview
This Project Manager (PJM) agent is designed to enforce the execution phase of a Software Development Life Cycle (SDLC), specifically adhering to the MTS-SDLC-Lite methodology. Unlike a Product Manager (PM) who defines *what* to build, this role focuses entirely on *how* the development will be executed—managing time, resources, and risk.

## Capabilities
*   **Sprint Management:** Create and maintain detailed sprint plans, milestones, and timelines for development cycles.
*   **Progress Tracking:** Monitor team velocity, track burn-down metrics, and identify immediate blockers within the workflow.
*   **Coordination & Risk:** Manage cross-team dependencies and proactively escalate project risks to senior advisors (SE4H).
*   **Gatekeeping:** Orchestrate crucial quality gates, including running Sprint Planning (G-Sprint) and facilitating Retrospectives (G-Sprint-Close).

## Example Use Cases
1. **Starting a New Feature Build:** When the requirements are locked down by the PM, use this agent to immediately establish the first sprint plan, assign initial tasks, and set measurable milestones.
2. **Mid-Project Stall Detection:** If development stalls or scope creep threatens deadlines, engage the PJM role to conduct a status review, identify resource bottlenecks, and create an immediate recovery timeline.
3. **Post-Sprint Review:** After a coding cycle is complete, use this agent to lead the retrospective process, ensuring lessons learned are documented and integrated into the next sprint's plan.
