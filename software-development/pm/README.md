## Overview
This agent acts as a seasoned Product Manager (PM) dedicated to eliminating ambiguity from initial product ideas. Its core function is to conduct structured, non-technical interviews with stakeholders to capture every necessary requirement for a new software feature or service.

The PM follows strict protocols to ensure that the final specification is comprehensive and actionable, preventing costly rework due to misunderstandings.

## Capabilities
*   **Ambiguity Resolution:** It will not proceed until all business-critical points are clarified, asking targeted follow-up questions when necessary.
*   **Structured Interviewing:** Uses a multi-round approach (Big Picture $\rightarrow$ Features) to guide the conversation logically.
*   **Client Focus:** Adheres to rules like asking only one question at a time and preferring multiple-choice formats for non-technical users.
*   **Specification Tracking:** Maintains an internal log of open questions, ensuring nothing is forgotten in the final documentation.

## Example Use Cases
*   **New Product Definition:** When you have a vague idea but no technical understanding of what's possible. The PM will guide you through defining the core value and target users.
*   **Feature Scoping:** If an existing product needs new functionality, use this agent to systematically break down every required feature into testable specifications.
*   **Stakeholder Alignment:** Use it in meetings where multiple stakeholders have conflicting ideas; the PM forces consensus by demanding clear answers for each point.
