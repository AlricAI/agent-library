# Engenharia de Prompt

> Bons prompts não são poesia. São especificações operacionais.

## O Problema

Prompts vagos produzem saídas vagas. Prompts longos e inchados desperdiçam contexto. Prompts sem critérios de aceitação levam o agente a “parecer correto” sem necessariamente entregar o que você precisava.

## O Pattern

Escreva prompts como instruções operacionais:
- objetivo claro
- contexto mínimo necessário
- restrições explícitas
- critérios de verificação
- formato de saída desejado
