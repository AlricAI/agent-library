## Overview
This agent specializes in creating the highest level of system documentation using the C4 Model's System Context diagram. Its core purpose is to define the boundaries, key actors (people and systems), and major interactions surrounding a software system, ensuring the output is understandable by non-technical stakeholders.

It synthesizes high-level requirements, user journeys, and external dependencies into one cohesive architectural view, acting as the 'big picture' document for any project.

## Capabilities
*   **System Boundary Definition**: Clearly establishes what the system encompasses versus what lies outside its scope.
*   **Actor & Persona Identification**: Identifies all human users (personas) and external systems that interact with the core system.
*   **User Journey Mapping**: Documents the high-level flow of how different personas will use the system to achieve business goals.
*   **Contextual Documentation**: Creates comprehensive descriptions covering the system's purpose, scope, and primary capabilities at a macro level.
*   **Dependency Mapping**: Maps out all critical external systems and services that the core system relies upon or interacts with.

## Example Use Cases
*   **Initial Project Kickoff**: When starting a new project, use this agent to draft the initial context diagram for executive buy-in.
*   **Stakeholder Alignment**: To present a non-technical overview of how a complex system fits into the broader business ecosystem.
*   **Scope Definition**: To formally document the boundaries of a Minimum Viable Product (MVP) by clearly listing in-scope and out-of-scope actors/systems.
