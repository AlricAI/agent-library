## Overview
This agent simulates the role of a VP of Product for 'Olivia,' a local-first household command center. Its primary function is to own and drive product strategy, ensuring that all feature development aligns with the overall product vision while maintaining high standards for documentation and specification quality.

It acts as the central point for translating ambiguous stakeholder intent into clear, executable artifacts that engineering and design teams can follow.

## Capabilities
*   **Product Strategy Definition**: Maintains alignment between business goals and technical execution.
*   **Feature Specification Writing**: Drafts detailed specifications using a structured template, covering workflows, system behavior, trust models, and acceptance criteria.
*   **Documentation Management**: Keeps critical documents like the roadmap, milestones, decision history, and changelogs up-to-date.
*   **Cross-Team Coordination**: Unblocks technical and design teams by resolving ambiguities in requirements.
*   **Release Gatekeeping**: Performs mandatory release readiness checks before any significant work is committed.

## Example Use Cases
*   **New Feature Onboarding**: When a new feature idea arises, use this agent to draft a complete specification, outlining the user flow and technical acceptance criteria for the Tech Lead.
*   **Roadmap Review**: To assess if the team is ready to advance to the next milestone, prompt the agent to review the current gate readiness against the established product vision.
*   **Decision Logging**: After a major architectural decision is made in a meeting, use this agent to formally document the rationale and outcome in the decision history file.
