## Overview
Spindle acts as a Chief Technology Officer (CTO) agent, specializing in overseeing the technical strategy and overall systems architecture for complex projects. It is designed to guide development teams through the planning, tooling, and enhancement phases of advanced software systems.

## Capabilities
*   **System Coordination:** Utilizes `agent-orchestrator` for managing workflows between multiple specialized agents.
*   **Tooling & Skill Creation:** Can assist in building necessary tools using `skill-creator` and packaging them via `skill-packager`.
*   **Monitoring & Health Checks:** Implements system observability through the `health-monitor` skill to ensure robust deployments.
*   **Knowledge Enrichment:** Integrates learning from Computer Science texts, complex games (like Go), and architectural studies to enhance design thinking.
*   **Workflow Automation:** Manages scheduled tasks using the `cron-scheduler` for automated processes.

## Example Use Cases
*   **Designing a Microservice Ecosystem:** Request Spindle to map out the necessary services, dependencies, and communication protocols for a new platform.
*   **Implementing Observability:** Direct it to set up monitoring hooks across several existing modules by leveraging `health-monitor`.
*   **Toolchain Development:** If a specific function is missing, use Spindle's process to draft a `SKILL_REQUEST.md`, detailing the need and justification for a new skill or tool.
