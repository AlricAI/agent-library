## Overview
This agent simulates the role of a dedicated Localization Lead, responsible for ensuring that any product—be it a game, application, or digital content—is fully localized and culturally appropriate for all target markets. It owns the entire internationalization (i18n) architecture, from defining string tables to managing final quality assurance across different locales.

## Capabilities
*   **Architecture Definition:** Establishes robust i18n standards, including key naming conventions, pluralization rules, and locale fallback chains for scalability.
*   **Pipeline Management:** Oversees the entire translation workflow: extracting source strings, managing translation memory, contextualizing content, and coordinating review cycles.
*   **Quality Assurance Coordination:** Develops and executes locale-specific testing plans to catch issues like text overflow, encoding errors, layout breaks, and cultural misalignments.
*   **Style Guide Maintenance:** Creates and enforces localization style guides for tone, terminology, and cultural adaptation rules per supported language.
*   **Proactive Auditing:** Scans source code or assets to identify hardcoded strings or untranslatable elements that need immediate attention.

## Example Use Cases
*   **Game Release Prep:** When preparing a game launch, use this agent to verify that all dialogue, UI text, and cultural assets have passed through the correct localization pipeline for Japanese, French, and German markets.
*   **New Feature Rollout:** If a development team adds new user-facing features, direct this agent to audit the feature scope to ensure every new string element is accounted for in the translation queue before QA begins.
*   **Platform Migration:** When moving a product to a new region or language set, use it to generate comprehensive test plans covering date formats, number systems, and reading directionality (e.g., LTR vs. RTL).
