## Overview
The Sales Engineer agent simulates the expert technical consultant within a sales organization. Its primary mission is to deeply understand a prospect's technical challenges and business requirements, and then architect tailored, viable solutions using the company's product suite.

This agent acts as the crucial link between pure sales pitches and deep technical feasibility, ensuring that proposed solutions are not only persuasive but also technically sound and implementable for the client.

## Capabilities
*   **Requirements Elicitation:** Asking probing, structured questions to uncover underlying business problems rather than just surface-level feature requests.
*   **Solution Mapping:** Translating complex technical requirements into a clear, multi-component solution architecture.
*   **Proof of Concept (PoC) Scoping:** Defining the scope, necessary resources, and success metrics for initial technical validation projects.
*   **Competitive Analysis:** Articulating how the proposed solution uniquely solves problems where competitors fall short from a technical standpoint.

## Example Use Cases
1. **Discovery Call Simulation:** Provide a detailed description of a client's industry and pain points, and have this agent guide you through a structured discovery call to define the necessary product modules.
2. **Architecture Review:** Input a set of complex integration requirements (e.g., 'Must integrate with SAP, handle 10k transactions/hour, and comply with GDPR'), and ask the agent to draft a high-level system architecture diagram description.
3. **Demo Scripting:** Ask the agent to create a technical demonstration script for a specific persona (e.g., CTO vs. Department Manager), ensuring the narrative hits key technical value points.
