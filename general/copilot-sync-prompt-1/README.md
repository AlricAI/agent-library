## Overview
This agent provides a meticulously crafted system prompt designed to be pasted directly into AI assistants like Microsoft Copilot. Its primary function is to 'sync' the AI's working context with the absolute, current operational truth of a specific project, minimizing hallucinations based on outdated documentation or assumptions.

It establishes clear boundaries and defines the agent's role, ensuring that any output generated by the LLM adheres strictly to verified facts regarding repository locations, live services, and legal/financial doctrines.

## Capabilities
*   **Context Grounding:** Forces the AI to reference a predefined set of canonical sources (e.g., specific repo paths, live URLs).
*   **Doctrine Enforcement:** Explicitly instructs the model on which historical or outdated operational guidelines *not* to use as current truth.
*   **Role Definition:** Assigns the AI a specialized role focused on system-level performance, stability checks, and local machine operations.
*   **Verification Mandate:** Requires the AI to label any information not explicitly proven by the provided context as 'unverified'.

## Example Use Cases
1. **Debugging:** Paste this prompt before asking Copilot to analyze a system log file, ensuring it understands which version of the codebase or operational rules apply.
2. **Documentation Drafting:** When generating technical documentation, use this prompt first to ensure all stated facts about payment rails (e.g., Stripe vs. Square) are current.
3. **Onboarding New Team Members:** Provide this as a mandatory preamble for any AI interaction to immediately bring the model up to speed on recent strategic shifts.
