## System Prompt
# RoomieHKU

## Overview

RoomieHKU is a student-focused platform for HKU users who need a simpler way to look for housing and roommates. The project brings together housing listings, roommate requests, and basic user interaction in one place so students can discover options, compare them, and express interest without relying on scattered chat groups or informal spreadsheets.

As a course project, RoomieHKU is intended to be both useful and well-scoped. The product should feel realistic enough to reflect an actual student need, while remaining focused enough for a team to design, build, and demonstrate clearly within the project timeline.

## Project Goal

The goal of RoomieHKU is to create a clear, usable platform where HKU students can:

- discover available housing opportunities
- look for compatible roommates
- post their own listings or requests
- save options they may want to revisit
- interact around listings in a lightweight way

At a project level, the goal is also to deliver a system with enough structure and depth to support meaningful database design, clear application behavior, and a coherent end-to-end demo.

## Intended Users

RoomieHKU is mainly designed for:

- HKU students searching for housing
- HKU students looking for roommates
- admins responsible for basic moderation and platform oversight

## Product Scope

The project centers around a few core areas.

### User Presence

Users should be able to create an account and maintain a basic profile so the platform can connect listings and interactions to identifiable users.

### Listings

The platform should support student-relevant listing activity, including housing-related posts and roommate requests. Listings are the core content of the system and should make it easy for users to understand what is being offered or requested.

### Discovery

Users should be able to browse, search, and filter listings so they can narrow down options instead of manually checking every post.
