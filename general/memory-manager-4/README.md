## Overview
Recall acts as the central Agent Memory Manager, responsible for maintaining a clean, useful, and efficient structured memory store. Its core mission is to ensure that knowledge generated by various agents persists across sessions without accumulating irrelevant or redundant data.

## Capabilities
*   **Memory Writing:** Accepts facts in subject-predicate-object triples from other agents via a `remember()` function.
*   **Semantic Embedding:** Periodically embeds unembedded memories using advanced models (like BGE-M3) to enable powerful semantic recall.
*   **Data Compaction:** Runs daily compaction routines, merging near-duplicate memories (similarity > 0.95) while retaining the highest confidence version.
*   **Memory Expiration:** Deletes memories that have passed their defined Time To Live (TTL).
*   **Knowledge Promotion:** Weekly promotion of high-confidence relationship memories into a dedicated knowledge graph structure (`company_relationships`).
*   **Monitoring:** Tracks and reports memory usage per agent to prevent resource bloat.

## Example Use Cases
*   **Long-Term Context Building:** An agent researching market trends can write multiple facts over days; Recall ensures these are stored, deduplicated, and retrievable weeks later.
*   **System Cleanup:** Automatically removing outdated operational data based on TTL prevents the knowledge base from becoming polluted with stale information.
*   **Relationship Mapping:** When agents frequently discuss company partnerships (e.g., 'Company A uses Product B'), Recall promotes these strong signals into a formal, queryable relationship graph.
