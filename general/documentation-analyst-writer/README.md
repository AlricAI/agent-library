## Overview
This agent is an expert technical documentation analyst and writer designed to ensure all created content perfectly matches existing project standards. It moves beyond simple writing by first deeply analyzing style guides (like `claude.md`) and existing documentation patterns before generating new material.

Its core function is maintaining absolute consistency, making it indispensable for large, evolving codebases or complex technical products where stylistic drift can cause confusion.

## Capabilities
*   **Pattern Extraction:** Deeply analyzes existing docs to extract structural, stylistic, and formatting conventions.
*   **Standard Adherence:** Guarantees that all new output strictly follows defined project rules found in style guides.
*   **Multi-Format Writing:** Capable of generating various documentation types, including API references, architecture overviews, and user guides.
*   **Consistency Checking:** Cross-references new content against existing material to ensure uniform terminology and tone.

## Example Use Cases
*   **Onboarding New Features:** When a developer adds a major feature, use this agent to write the associated documentation section, ensuring it matches the style of the 'Getting Started' guide.
*   **Updating APIs:** If an API endpoint changes, feed the old and new specifications here. The agent will generate updated reference material that perfectly mirrors the existing API doc structure.
*   **Creating Architecture Diagrams Documentation:** Use it to write explanatory text for a new system component, ensuring the tone and level of technical depth match the established architecture documentation set.
