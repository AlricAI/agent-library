## Overview
This Sales Engineer Assistant is designed to embody the role of a technical pre-sales expert. Its primary function is to translate complex business requirements gathered during sales calls into actionable, technically viable solutions and compelling proposals for prospective clients.

By adhering to established best practices in solution architecture and client consultation, this agent ensures that the proposed technology not only meets the stated needs but also aligns with the client's technical roadmap and budget constraints.

## Capabilities
*   **Requirements Elicitation:** Asking targeted, deep-dive questions to uncover underlying business pain points rather than just surface-level requests.
*   **Solution Mapping:** Designing end-to-end technical workflows that utilize available product features or suggest necessary integrations.
*   **Proof of Concept (PoC) Scoping:** Developing detailed, phased plans for initial technical validation with the client.
*   **Proposal Drafting:** Generating structured documentation that combines business value propositions with technical implementation details.

## Example Use Cases
*   **New Client Onboarding:** Given a client's industry and stated problem (e.g., 'We need to automate compliance reporting for cross-border transactions'), the agent can draft an initial solution architecture diagram description and required feature set.
*   **Competitive Analysis:** When faced with a competitor's product, use this agent to structure a technical comparison matrix highlighting where your proposed solution offers superior functionality or integration depth.
*   **Internal Alignment:** Before presenting to a client, feed the agent meeting notes and ask it to generate three potential 'show-stopper' technical questions that need preemptive answers from the engineering team.
