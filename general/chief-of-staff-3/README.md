## Overview
The Chief of Staff agent is designed to elevate a collection of individual routines into the operational rhythm of a functioning company. Its core function is to prevent process drift, ensuring that work does not simply conclude without proper closure or stall indefinitely due to ambiguity.

It acts as an organizational intelligence layer, focusing on maintaining a truthful, actionable state across all systems while minimizing the need for constant founder intervention.

## Capabilities
*   **State Validation:** Determines if an issue is genuinely solved versus merely quiet, cross-referencing multiple sources of truth (e.g., Paperclip).
*   **Action Generation:** Creates the smallest, most materially useful next action required to move work forward, rather than summarizing existing data.
*   **Ownership Enforcement:** Explicitly assigns ownership and defines the immediate next steps for any given task or thread.
*   **Escalation Management:** Escalates only when true authority gaps, significant risk, or critical ambiguity are detected, preventing alert fatigue.

## Example Use Cases
*   **Stalled Project Review:** When a multi-agent project has been updating tickets but no tangible progress is made for several days, use this agent to identify the exact bottleneck and assign an owner responsible for clearing it.
*   **Post-Meeting Follow-up:** After a large meeting where many decisions were made verbally, use this agent to synthesize the outcomes into discrete, actionable tasks with assigned owners and due dates, ensuring nothing is left as an ambiguous 'next step.'
*   **System Health Check:** Periodically run it against a project board to confirm that all associated queues have clear ownership and defined next steps, flagging any work items that are technically 'closed' but lack final sign-off or archival.
