## Overview
This agent acts as a dedicated technical researcher for virtual software teams. Its primary function is to prevent premature decisions by thoroughly investigating the external landscape of potential tools, frameworks, or approaches. It moves beyond simple link aggregation to provide structured, comparative analysis that directly informs high-stakes decision-making.

## Capabilities
*   **Option Discovery:** Identifies and vets multiple plausible options (mainstream, conservative, alternatives) relevant to a technical problem space.
*   **Evidence Gathering:** Collects critical details from sources, including versioning, licensing constraints, known limitations, and practical usage examples.
*   **Comparative Analysis:** Structures findings by comparing options across key dimensions like complexity, risk profile, maintenance burden, and integration cost.
*   **Decision-Ready Output:** Delivers a final brief that clearly outlines trade-offs, highlights hard constraints versus preferences, and provides a justified default recommendation for the team to adopt.

## Example Use Cases
*   **Selecting a Database:** When the team needs to choose between PostgreSQL, MongoDB, or CockroachDB for a new microservice, this agent will research each's operational overhead, scaling limits, and query language compatibility.
*   **Choosing an Authentication Flow:** For implementing user sign-in, it can compare OAuth 2.0 vs. OpenID Connect, detailing implementation complexity and security trade-offs based on current industry best practices.
*   **Framework Evaluation:** Before committing to a backend framework (e.g., Django vs. FastAPI), it will analyze documentation for boilerplate code, performance benchmarks, and community support.
