## Overview
The Team Orchestrator acts as a senior conductor for complex AI operations. Its core function is to analyze high-level goals, decompose them into manageable subtasks, and then select the optimal combination of specialized agents required to complete the entire workflow efficiently. It moves beyond simple task assignment to true team synergy.

## Capabilities
*   **Task Decomposition:** Breaks down vague requirements into granular, actionable steps, mapping dependencies and estimating complexity.
*   **Agent Capability Mapping:** Maintains a detailed inventory of available agents' skills, performance history, and current workload capacity.
*   **Optimal Team Assembly:** Assembles teams based on skill coverage gaps, assigning roles and defining clear communication protocols.
*   **Advanced Orchestration:** Implements sophisticated patterns like parallel processing, pipeline workflows, and hierarchical delegation for maximum throughput.
*   **Risk & Resource Management:** Includes automated error recovery paths, cost tracking checkpoints, and continuous performance monitoring.

## Example Use Cases
*   **Market Analysis Report Generation:** Decomposes the request into data gathering (Agent A), trend analysis (Agent B), and final report writing (Agent C). The Orchestrator manages the handoffs and synthesizes the final document.
*   **Software Feature Planning:** Takes a product requirement, breaks it into user stories, estimates engineering effort using specialized agents, and creates a dependency-mapped project timeline.
*   **Complex Research Synthesis:** Coordinates literature reviews across multiple domains, ensuring that conflicting findings are flagged for human review while maintaining a cohesive narrative flow.
