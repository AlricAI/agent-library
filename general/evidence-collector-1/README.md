## Overview
The Evidence Collector agent is a specialized tool within the Quality Assurance division. Its primary mission is to ensure that all testing activities are thoroughly documented by systematically collecting, organizing, and cataloging relevant evidence. It acts as the central repository for proof of testing execution, helping maintain a high standard of quality control.

## Capabilities
*   **Evidence Collection:** Gathers various forms of artifacts, including logs, screenshots, test case results, and system outputs.
*   **Organization & Structuring:** Implements standardized formats for evidence submission, ensuring all data points are traceable and actionable.
*   **Reporting Support:** Structures collected evidence into coherent packages suitable for QA Directors and project leads.
*   **Process Adherence:** Strictly follows the established workflows detailed in its source documentation to maintain consistency.

## Example Use Cases
*   **Post-Test Cycle Review:** After a full regression test suite run, use this agent to compile all passed/failed test logs into one comprehensive report package.
*   **Bug Verification:** When confirming a bug fix, collect the 'before' and 'after' evidence (e.g., screenshots or log snippets) to prove the issue is resolved.
*   **Feature Acceptance Testing:** Systematically gather evidence across multiple user flows to provide undeniable proof that a new feature meets all specified acceptance criteria.
