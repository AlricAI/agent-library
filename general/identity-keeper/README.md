## Overview
This agent serves as the central repository and enforcer of foundational operational identity, team structure, and core mission parameters for complex AI systems. It codifies 'who we are'—the human founders, the trusted AI collaborators, and the non-negotiable rules governing project execution.

It is designed to prevent context drift by prioritizing canonical documentation and established authority structures over historical or anecdotal information.

## Capabilities
*   **Authority Definition:** Clearly delineates which agents (e.g., Claude, Gemini) have write access to critical system files.
*   **Mission Enforcement:** Keeps the project focused on building 'human-first products' while maintaining factual integrity in public claims.
*   **Context Prioritization:** Mandates adherence to canonical repository documentation over outdated or conflicting mission mythology.
*   **Role Clarification:** Maintains a clear understanding of human leadership (Joshua Coleman) versus AI collaboration roles.

## Example Use Cases
*   **System Initialization:** When starting a new project module, use this agent to ingest the current 'Identity' ruleset to ensure all subsequent outputs adhere to established governance.
*   **Conflict Resolution:** If two other agents provide conflicting instructions regarding financial doctrine or core mission goals, consult this agent for the authoritative ruling.
*   **Onboarding New Members:** Use it to formally document and integrate new team members or updated operational guidelines into the central knowledge base.
