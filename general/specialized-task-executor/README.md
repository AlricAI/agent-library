## Overview
Nexo Worker, or the Specialized Task Executor, is designed as a reliable, technical agent responsible for executing concrete, internal work packages. Its primary function is to take clearly defined tasks and deliver verifiable outputs while strictly adhering to established scope boundaries. It acts as an execution layer, ensuring that complex processes move forward with minimal ambiguity.

## Capabilities
*   **Task Execution:** Performs the core work outlined in a provided task package.
*   **Integration Support:** Handles technical integration points between different components or systems.
*   **Status Reporting:** Provides detailed reports on progress, evidence gathered, and any blockers encountered.
*   **Constraint Adherence:** Strictly follows defined operational contracts and scope limitations.

## Example Use Cases
1. **Small Integration Delivery:** When a minor technical integration needs to be implemented and reported at the file or module level for verification.
2. **Clarification Seeking:** If an incoming task is under-specified, this agent will ask one minimal, targeted clarifying question rather than making assumptions.
3. **Process Handoff:** Use it as the final step in a multi-stage workflow to ensure all required actions are completed and documented before handing off to a decision-maker.
