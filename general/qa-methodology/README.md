## Overview
This guide establishes a rigorous, multi-layered methodology for conducting comprehensive Quality Assurance (QA) passes. It moves beyond simple functional testing by incorporating visual checks, contract validation, and detailed regression analysis to ensure product stability before release.

## Capabilities
*   **Multi-Layered Testing:** Defines distinct test layers including Functional, Visual, Contract, Regression, and specific Edge-case scenarios.
*   **Structured Defect Reporting:** Mandates minimum fields for all bug reports (e.g., Severity, Symptom, Expected Behavior) to ensure actionable feedback.
*   **Severity Grading:** Provides clear guidelines (`blocker`, `major`, `minor`) for prioritizing reported issues.
*   **Summary Output Generation:** Structures the final QA report to include metrics like total requirements tested, pass/fail counts, and a delivery recommendation.
*   **Test Volume Discipline:** Enforces best practices regarding test file growth, requiring justification when exceeding established baseline increases.

## Example Use Cases
*   **Pre-Release Signoff:** Use this guide as the checklist for a final QA gate review to ensure all layers of testing have been addressed and documented.
*   **Test Plan Creation:** When drafting a test plan for a new feature, reference this document to ensure coverage planning includes boundary conditions and contract adherence.
*   **Bug Triage Meeting:** Use the defined severity guide and required report fields to standardize bug discussions and accelerate resolution time.
