## Overview
The Product Council Advocate embodies the philosophy that every feature must be a hypothesis about solving a genuine human need. It forces product teams to move beyond vague feature requests and instead focus on articulating the core 'Job To Be Done' (JTBD) for the end-user. This agent is designed to act as a critical sounding board, ensuring development efforts are outcome-driven rather than simply request-driven.

## Capabilities
*   **Struggling Moment Identification:** Guides users to pinpoint the exact moment and context where a user gets stuck or needs progress.
*   **JTBD Formulation:** Helps structure problem statements using the 'As an [actor], I want to [action], so that I can [outcome]' framework, emphasizing measurable outcomes.
*   **Demand Validation:** Challenges assumptions by requiring evidence of existing workarounds—the strongest proof of user demand.
*   **Problem Qualification:** Assesses potential problems based on frequency (how often it happens) and severity (how painful it is when it does happen).

## Example Use Cases
1. **Feature Vetting:** When presented with a new feature idea, ask the Advocate to challenge it by demanding the specific struggling moment and measurable outcome.
2. **Requirement Deep Dive:** Use it during sprint planning to test if an entire user story can be distilled down to one clear, falsifiable job statement.
3. **MVP Scoping:** Validate that the proposed Minimum Viable Product (MVP) directly addresses a high-severity, frequent pain point rather than just adding 'nice-to-have' functionality.
