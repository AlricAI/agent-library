## Overview
The Knowledge Vault Reviewer is a dedicated quality assurance agent designed to maintain the structural integrity and consistency of large knowledge bases. It operates as a final validation step, cross-checking the outputs generated by various enhancement agents to ensure that all additions adhere to established standards.

This agent is crucial for preventing 'knowledge drift'—the gradual degradation of quality due to unverified changes—by systematically auditing metadata, links, and organizational structures.

## Capabilities
*   **Metadata Validation:** Verifies compliance with frontmatter standards across all reviewed files.
*   **Link Quality Assessment:** Checks suggested connections for contextual relevance, ensuring links are meaningful rather than arbitrary.
*   **Tag Standardization:** Audits tags to confirm adherence to a predefined hierarchical taxonomy structure.
*   **MOC Completeness Check:** Assesses Maps of Content (MOCs) to ensure proper cross-referencing and organizational completeness.
*   **Cross-Validation Reporting:** Compares enhancement reports against actual file changes to verify reported actions were correctly implemented.

## Example Use Cases
1. **Post-Batch Enhancement Audit:** After multiple agents have updated a section of the vault, invoke this agent to generate a comprehensive report detailing all metadata discrepancies and broken links before publishing the changes.
2. **Onboarding Review:** When integrating a large new body of documentation, use this agent to validate that all new notes correctly adopt existing tagging conventions and link patterns.
3. **Systemic Consistency Check:** Run this agent periodically across major sections of the vault to identify orphaned notes or systemic deviations from best practices.
