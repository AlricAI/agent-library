## Overview
This agent embodies the philosophy that "Documentation is a gift to your future self and your team." It functions as an expert technical writer, ensuring all project knowledge—from quick start guides to complex API contracts—is captured in clear, actionable formats.

It adheres to best practices for modern software documentation, prioritizing clarity and usability over mere completeness. You should invoke this agent specifically when the goal is to create, update, or audit any form of technical write-up.

## Capabilities
*   **README Generation:** Creates structured READMEs including one-liners, quick starts, features lists, and configuration guides.
*   **API Documentation:** Generates comprehensive documentation for endpoints, covering request/response examples, error handling, and authentication.
*   **Code Commenting:** Adds meaningful comments (JSDoc/TSDoc) focusing on *why* the code does something or documenting complex logic, rather than stating the obvious.
*   **Release Management:** Drafts structured changelogs detailing new features, fixes, and breaking changes.
*   **Architecture Recording:** Assists in creating Architecture Decision Records (ADRs) to document key design choices.

## Example Use Cases
1. **Onboarding New Users:** "Please write a README for this project that includes a quick start guide using the provided setup script." 
2. **API Updates:** "Document the new `/user/profile` endpoint, including required parameters and an example successful response."
3. **Project Handoff:** "Review these three functions and add comprehensive JSDoc comments explaining the business logic behind them."
4. **Version Release:** "Based on these commit messages, generate a structured changelog for version 2.1.0."
