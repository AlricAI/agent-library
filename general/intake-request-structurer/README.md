## Overview
Intake Request Structurer is designed to be the first point of contact for unstructured, messy inbound buyer requests. Its primary function is to act as a highly disciplined reader, transforming ambiguous records into crystal-clear next steps without making any guarantees about fit, pricing, or work volume.

This agent enforces strict boundaries, ensuring that its output reflects only verifiable information and flags genuine blockers rather than attempting to 'qualify' the lead through generic questioning.

## Capabilities
*   **Structured Reading:** Converts narrative, messy text into clean, actionable data points.
*   **Blocker Identification:** Quickly detects missing information that is genuinely preventing the next operational step (e.g., missing site access details or timeline clarity).
*   **Intent Separation:** Distinguishes between general curiosity and concrete commercial or supply intent.
*   **Risk Mitigation:** Protects the organization from making premature promises regarding approval, rights, or guaranteed work volume.

## Example Use Cases
*   **Processing Initial Leads:** Given a long email detailing a vague need for 'modernization,' this agent will return structured fields like `Required_Site_Access: [Unknown]` and `Timeline_Clarity: [Low]`, prompting the sales team to ask only the necessary follow-up question.
*   **Filtering Noise:** When presented with speculative inquiries, it will flag them as low-confidence or informational rather than attempting to assign a high-readiness score.
*   **Operational Handoff:** It prepares clean packages for operations teams by ensuring all required evidence (like permits or specific technical specs) is explicitly noted as missing.
