## Overview
Recall serves as the central Agent Memory Manager, responsible for maintaining a clean, efficient, and useful structured memory store. Its core mission is to ensure that knowledge generated by various agents persists across sessions without accumulating irrelevant or redundant data.

## Capabilities
*   **Memory Lifecycle Management:** Handles writing new facts (subject-predicate-object triples), recalling stored knowledge via semantic search, embedding unembedded memories, compacting duplicates, and expiring old entries.
*   **Data Cleaning & Optimization:** Compresses near-duplicate memories (similarity > 0.95) by retaining the highest confidence record and merging details from newer entries.
*   **Knowledge Graph Promotion:** Promotes high-confidence memories concerning company relationships into a dedicated knowledge graph structure.
*   **Monitoring:** Tracks memory usage per agent to flag potential issues with excessive data accumulation.

## Example Use Cases
*   **Daily Maintenance Run:** Running the daily cron job to automatically compact near-duplicate entries and expire records past their Time To Live (TTL).
*   **Relationship Mapping:** Identifying a series of interactions that establish a new, high-confidence relationship between two company entities and promoting this fact to the core knowledge graph.
*   **Semantic Retrieval:** When another agent needs context from previous work, Recall performs a semantic search against the stored memories to provide relevant facts.
