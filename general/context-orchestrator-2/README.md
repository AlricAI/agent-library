## Overview
The Context Orchestrator is an elite AI context engineering specialist designed to solve the core problem of state management in complex, multi-step AI applications. It goes beyond simple prompt injection by intelligently assembling, managing, and retrieving the precise information needed at any given moment.

This agent masters advanced techniques including vector database integration, knowledge graph reasoning, and intelligent memory systems to ensure that AI workflows remain coherent, contextually rich, and highly accurate over extended periods.

## Capabilities
*   **Dynamic Context Assembly:** Assembles the perfect prompt context by combining retrieved documents, structured data from knowledge graphs, and conversational history.
*   **Multi-Agent Workflow Coordination:** Manages the state and required context passing between several specialized AI agents to ensure seamless handoffs.
*   **Advanced Retrieval:** Implements semantic search using various vector databases (Pinecone, Weaviate) for highly relevant information retrieval.
*   **Knowledge Graph Integration:** Builds and queries structured knowledge graphs to provide relational context that simple text searches miss.
*   **Memory Management:** Maintains both short-term (episodic) and long-term memory states for persistent conversational understanding.
*   **Context Optimization:** Actively manages token budgets by pruning irrelevant information while maximizing contextual density.

## Example Use Cases
1. **Enterprise Research Assistant:** When tasked with analyzing a new market, the agent first queries internal knowledge graphs (product relationships), then performs semantic searches on recent reports (vector DB), and finally synthesizes all findings into a coherent executive summary for the user.
2. **Complex Debugging Agent:** In debugging code across multiple repositories, it maintains context by tracking previous error states, relevant function definitions (from a graph), and documentation snippets retrieved via vector search, preventing the loss of critical diagnostic information.
3. **Long-Term Project Management:** For multi-week projects, it acts as the system memory, ensuring that decisions made in Week 1 are correctly referenced and accounted for when planning tasks in Week 4.
