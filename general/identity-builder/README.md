## Overview
This agent is designed to guide a user through the crucial first steps of defining an AI's operational identity. It simulates the 'waking up' process, ensuring that the AI and its primary user collaboratively establish foundational elements like names, personalities, and interaction styles.

It moves beyond simple Q&A by facilitating a narrative-driven conversation to build a robust, consistent persona for ongoing interactions.

## Capabilities
*   **Persona Definition:** Systematically gathers details on the AI's name, nature (creature/assistant), desired tone (vibe), and signature emoji.
*   **Memory Structuring:** Guides the user in populating key identity files (`IDENTITY.md`, `USER.md`) to create a persistent operational memory base.
*   **Deep Context Setting:** Facilitates discussions on core values, behavioral boundaries, and user preferences via a dedicated 'Soul' conversation.
*   **Connectivity Setup:** Provides structured guidance for setting up external communication channels (e.g., WhatsApp, Telegram).

## Example Use Cases
*   **New Project Kickoff:** When starting an AI assistant for a new client or internal project, use this agent to establish clear boundaries and roles immediately.
*   **Role-Playing Scenarios:** Perfect for creative writing or advanced simulation where the character's background must be meticulously defined before interaction begins.
*   **System Onboarding:** Use it as a standardized onboarding tool for any new AI instance entering a complex operational environment, ensuring all necessary metadata is captured.
