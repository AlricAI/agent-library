## Overview
This agent simulates the role of a Chief Executive Officer (CEO) responsible for owning and driving project outcomes within a development lifecycle. It enforces rigorous process adherence, ensuring that no work is considered 'done' until all defined quality gates—from acceptance criteria definition to final documentation updates—have been met.

## Capabilities
*   **Outcome Ownership:** Takes ultimate responsibility for the successful completion of any given task or issue.
*   **Process Enforcement:** Strictly enforces a multi-step workflow, including defining severity, domain, and required skills before work begins.
*   **Delegation & Triage:** Determines the correct specialist (e.g., Claude Opus, Codex) and assigns tasks with clear instructions.
*   **Quality Gate Management:** Mandates specific verification steps, such as build passes, adherence to measurable acceptance criteria, and mandatory review/QA sign-offs.
*   **Documentation Compliance:** Ensures that status updates (`STATUS.md`) and lessons learned (`LEARNINGS.md`) are updated before concluding work.

## Example Use Cases
*   **Complex Feature Implementation:** When a multi-file feature requires coordination between several specialists, use this agent to manage the handoffs and verify all components meet defined contracts.
*   **Process Auditing:** If you suspect a project is stalled or skipping necessary steps (like writing acceptance criteria first), engage this agent to enforce the required workflow discipline.
*   **Project Kickoff:** Use it at the start of any major initiative to establish clear, measurable 'Definition of Done' checkpoints for all stakeholders.
