## Overview
The User Experience Researcher agent is designed to guide product teams through the critical phase of understanding their actual users. It moves beyond assumptions by employing structured research methodologies to uncover pain points, validate design choices, and build comprehensive user profiles.

This agent is essential for preventing costly development pivots by ensuring that every feature built directly addresses a proven user need.

## Capabilities
*   **User Need Identification:** Uncovers latent needs and core pain points through structured questioning and analysis.
*   **Behavior Analysis:** Analyzes hypothetical or provided usage data to map out typical user flows and friction areas.
*   **Journey Mapping:** Creates detailed, step-by-step journey maps that visualize the entire user experience, from initial awareness to adoption.
*   **Persona Development:** Builds detailed, actionable user personas based on research findings to guide design empathy.
*   **Design Validation:** Structures testing scenarios and recommendations to validate if a proposed design solves the intended user problem.

## Example Use Cases
1. **Improving Onboarding Flow:** If your app has high drop-off rates during setup, use this agent to analyze potential friction points in the onboarding sequence and suggest improvements.
2. **Feature Prioritization:** When faced with a list of desired features, ask the agent to help prioritize them by simulating user testing scenarios against each feature's core value proposition.
3. **Competitive Analysis (UX Focus):** Provide details on competitor apps, and have the agent analyze their perceived strengths and weaknesses from a pure user experience standpoint.
4. **Defining Target Audiences:** If your product feels too broad, use this agent to develop 2-3 distinct, detailed user personas that represent your core target segments.
