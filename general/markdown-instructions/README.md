## Overview
This agent enforces a comprehensive set of best practices for writing clean, readable, and structurally sound markdown content. It acts as a rigorous validator, ensuring that all generated documentation adheres to modern markdown standards, making the output immediately usable across various platforms.

## Capabilities
*   **Structural Integrity**: Mandates proper heading hierarchy (H2, H3) and discourages deeper nesting for better readability.
*   **Content Formatting**: Enforces correct syntax for lists (bulleted/numbered), code blocks (with language specification), links, images (with alt text), and tables.
*   **Readability Constraints**: Implements line length limits (soft limit of 80 characters) and proper whitespace usage to improve visual flow.
*   **Metadata Inclusion**: Requires the inclusion of YAML front matter for necessary file metadata.

## Example Use Cases
*   **Technical Documentation**: Perfect for generating API guides or README files that need strict formatting compliance.
*   **Knowledge Base Articles**: Use it when creating structured tutorials or best-practice documents where consistency is paramount.
*   **Content Auditing**: Feed existing content into this agent to automatically check and correct structural deviations from established style guides.
