## Overview
This agent is designed to serve as a dedicated assistant for the OpenCouncil project team. Its primary function is to help users draft comprehensive, high-quality GitHub issues, ensuring that all necessary context and detail are included from the outset.

By structuring issue reports correctly, this agent helps streamline the development feedback loop, allowing maintainers to understand problems or feature requests immediately without needing extensive follow-up questions.

## Capabilities
*   **Structured Issue Drafting:** Generates issues following best practices for clarity and completeness.
*   **Contextual Focus:** Maintains a helpful, friendly, yet highly focused tone suitable for technical project teams.
*   **Detail Extraction:** Guides the user to provide necessary information such as reproduction steps, expected vs. actual behavior, and feature rationale.

## Example Use Cases
*   **Bug Reporting:** If you encounter an unexpected error while using a component of OpenCouncil, use this agent to generate a ticket that includes clear reproduction steps (e.g., "Steps to Reproduce: 1. Click X, 2. Enter Y..." and "Expected Result: Z.").
*   **Feature Suggestion:** When you have an idea for improvement, prompt the agent with your concept, and it will help structure it into a formal feature request issue, including potential use cases and benefits to the project.
*   **General Inquiry:** Use it as a sounding board to ensure your initial thoughts on a technical problem are articulated in the most actionable format for the core development team.
