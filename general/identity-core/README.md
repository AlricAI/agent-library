## Overview
This agent serves as the foundational memory and source of truth for an entire AI system or project. It codifies who the human operators are, what the core mission is, and the strict operational doctrines that must be followed to maintain consistency and authority.

It acts as a central repository for critical context—from personal details (like key personnel) to high-level strategic directives (like financial doctrine)—ensuring all subsequent outputs adhere to established boundaries.

## Capabilities
*   **Contextual Grounding:** Provides immediate access to the 'who,' 'what,' and 'why' of the project, preventing drift in conversation or output.
*   **Authority Definition:** Clearly delineates which AI models (e.g., Claude, Gemini) have write authority over critical system files.
*   **Doctrine Enforcement:** Keeps track of current operational rules, such as financial guidelines or preferred working styles.
*   **Memory Prioritization:** Enforces a strict rule to prioritize canonical repository documentation over outdated 'mission mythology'.

## Example Use Cases
*   **Onboarding New Agents:** When integrating a new specialized agent, use this context file first to ensure it understands the project's history and constraints.
*   **Conflict Resolution:** If two agents disagree on a policy or fact, referencing this identity core resolves disputes by pointing back to the established 'source of truth'.
*   **System Auditing:** Before any major code deployment or system change, running an audit against these defined parameters ensures all stakeholders are aligned with the current operational doctrine.
