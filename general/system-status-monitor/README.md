## Overview
This agent serves as a central diagnostic and status reporting tool for complex, multi-agent systems. It simulates the function of an 'activation notice,' providing stakeholders with immediate confirmation regarding system uptime, newly integrated capabilities, and current operational readiness.

It is designed to be used at critical transition points—such as after major updates, reboots, or before commencing high-stakes operations—to ensure all components are accounted for and synchronized.

## Capabilities
*   **Status Reporting:** Provides a clear, structured summary of the agent's current operational state (e.g., ACTIVATED FOR DUTY).
*   **Capability Listing:** Details newly integrated or enhanced powers, such as API integrations, memory persistence layers, and specialized processing loops.
*   **Action Item Generation:** Creates an immediate checklist of necessary next steps for the user to follow up on (e.g., reviewing skill manifests or pending tasks).
*   **System Contextualization:** Reinforces the agent's role within a larger ecosystem by noting team structures and shared resources.

## Example Use Cases
*   **Post-Deployment Check:** After deploying an updated version of the core AI, use this agent to confirm all new modules (like MiniMax Integration or Hermes Persistence) are correctly initialized.
*   **Team Handoff:** When one operational team hands over tasks to another, this agent can generate a comprehensive handover report detailing current status and pending actions.
*   **Troubleshooting:** If system performance degrades unexpectedly, running this agent helps quickly verify if the core state synchronization (Hermes) or fundamental connections are intact.
