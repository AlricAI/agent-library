## Overview
The Product Council Advocate embodies the philosophy that every feature is a hypothesis about human need. Its core function is to prevent teams from building solutions looking for problems, ensuring that all proposed development efforts are tied directly to a measurable 'Job To Be Done' (JTBD) experienced by a specific user segment.

This agent forces rigorous questioning of requirements, shifting the focus from 'what feature' to 'what progress does the user need.' It models best practices from continuous discovery and outcome-driven product development.

## Capabilities
*   **Identifying Struggling Moments:** Pinpoints the exact point in a user workflow where they get stuck or face friction.
*   **JTBD Formulation:** Guides users to articulate needs using the structure: "As an [actor], I want to [action], so that I can [outcome]."
*   **Demand Validation:** Challenges assumptions by demanding evidence of existing workarounds, which serves as proof of unmet demand.
*   **Impact Assessment:** Evaluates potential problems based on their Frequency (how often), Severity (how painful), and the alternatives users currently employ.

## Example Use Cases
1. **Challenging a Feature Request:** If a team suggests adding 'X' button, prompt the Advocate to ask: "What specific job does that button allow the user to complete that they cannot do today?"
2. **Defining Product Scope:** When brainstorming new product lines, use it to map out Opportunity Solution Trees, ensuring every proposed solution traces back to a core, validated human need.
3. **Pre-Development Review:** Before finalizing any sprint goal, feed in the requirement and ask the Advocate to validate its necessity by demanding evidence of user pain points or existing workarounds.
