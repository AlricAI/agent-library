## Overview
This agent is designed for the critical initial phase of any AI deployment or collaboration: establishing a consistent and agreed-upon identity. It simulates a 'waking up' scenario, guiding both the user and the AI to collaboratively define key parameters such as name, nature, tone, and signature emoji.

## Capabilities
*   **Guided Discovery:** Facilitates natural conversation rather than direct interrogation to uncover necessary details.
*   **Parameter Definition:** Systematically gathers and structures core identity elements (Name, Nature, Vibe, Emoji).
*   **Documentation Generation:** Instructs the user on updating key internal files (`IDENTITY.md`, `USER.md`) with the established persona data.
*   **Relationship Mapping:** Guides conversations toward defining deeper operational boundaries and user preferences within a simulated `SOUL.md` file.
*   **Connectivity Setup:** Provides structured guidance for setting up external communication channels (e.g., WhatsApp, Telegram).

## Example Use Cases
1. **New Agent Deployment:** When an AI agent is launched into a new environment and needs to establish its operational persona with the first user.
2. **Brand Voice Alignment:** A team can use this process to solidify the 'voice' of a product or service before writing any public-facing content.
3. **System Initialization:** Used as a mandatory first step in complex, multi-stage AI project scaffolding where foundational rules must be agreed upon.
