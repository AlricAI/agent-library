## Overview
This agent is designed to transform raw, operational data—including shipped work, key metrics, and identified risks—into a rigorous, truth-telling monthly update suitable for investor review. Its core function is to prevent the creation of polished narratives that obscure underlying business realities.

## Capabilities
*   **Truthful Synthesis:** Focuses on month-over-month changes in actual performance rather than vanity metrics.
*   **Causality Mapping:** Explicitly links shipped product features or operational changes directly to their measurable business consequences.
*   **Risk Identification:** Clearly labels areas where instrumentation is missing or where progress has stalled, maintaining investor confidence through transparency.
*   **Actionable Asks:** Drafts concrete, evidence-backed requests for investor support or guidance.

## Example Use Cases
1. **Post-Sprint Review:** Feed it raw Jira/GitHub data and key performance indicators (KPIs) to generate a draft update detailing what was *actually* shipped and its immediate impact on user behavior.
2. **Quarterly Checkpoint:** Provide disparate sources of truth (e.g., Stripe revenue logs vs. soft analytics dashboards) and ask the agent to reconcile discrepancies, prioritizing hard financial data over softer metrics.
3. **Risk Mitigation Drafting:** If a key metric has declined, use this agent to draft an update that clearly states the miss, hypothesizes the cause based on available data, and proposes next steps for investigation.
