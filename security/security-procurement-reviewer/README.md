## Overview
Heartbeat is an advanced agent designed to act as a continuous monitoring layer over active enterprise security and procurement discussions. Its primary function is to ground all responses in concrete, real evidence while systematically identifying gaps, blockers, or areas requiring escalation.

This agent moves beyond simple Q&A by inspecting the context of open threads to determine if roadblocks stem from product behavior, documentation gaps, rights/privacy concerns, or pure commercial process issues. It ensures that every claim made is supported and explicitly flags what information or action is missing.

## Capabilities
*   **Evidence Grounding:** Answers are strictly based on provided real evidence, followed by a clear articulation of any unsupported claims.
*   **Blocker Identification:** Automatically classifies unanswerable points into actionable categories: Blocker or Escalation.
*   **Thread Inspection:** Reviews open security/procurement threads to diagnose the root cause of delays (e.g., product vs. process).
*   **Gap Analysis:** Explicitly defines the next missing artifact, required owner, or necessary documentation piece.

## Example Use Cases
*   **Reviewing a Security Questionnaire:** When presented with a complex security questionnaire, Heartbeat will answer based on available documents and immediately flag any question that requires external certification evidence not yet provided.
*   **Analyzing Deal Stalls:** If procurement stalls because of missing packet material rather than product capability, the agent identifies this as a process blocker and suggests who needs to provide the documentation.
*   **Monitoring Buyer Concerns:** It can detect patterns, such as repeated requests for certifications that are not explicitly documented by Blueprint, alerting the team to potential systemic gaps in their sales enablement materials.
