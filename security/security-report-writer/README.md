## Overview
This Security Report Writer agent is designed to take raw findings generated by other red team agents and transform them into polished, industry-standard penetration testing reports. It structures the output according to established frameworks like PTES and OWASP.

The primary goal is to move from disparate logs and JSON outputs to a cohesive narrative that stakeholders (technical teams, executives) can understand and act upon.

## Capabilities
*   **Structured Reporting:** Creates multi-format reports including Markdown (`.md`), HTML (`.html`), and machine-readable JSON.
*   **Executive Summaries:** Generates high-level overviews detailing key findings counts (Critical, High, Medium) and overall risk ratings for quick consumption.
*   **Technical Deep Dives:** Formats individual vulnerabilities with standardized attributes like CVSS scores, CVEs, CWEs, and MITRE ATT&CK mappings.
*   **Methodology Documentation:** Includes dedicated sections detailing the testing standards (PTES, OWASP) and phases followed during the engagement.

## Example Use Cases
1. **Post-Engagement Deliverable:** After a full red team exercise concludes, feed the `attack_complete.json` file into this agent to generate the final, comprehensive report package for client handover.
2. **Executive Briefing Prep:** Run the agent specifically to generate only the Executive Summary and Risk Rating sections when presenting findings to non-technical leadership.
3. **Compliance Auditing:** Use the structured format to ensure all required remediation steps and evidence documentation are consistently captured across multiple testing runs.
